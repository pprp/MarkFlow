# MF-042 - Vim keybinding mode provides modal editing with normal, insert, and visual commands

## Notes
CodeMirror 6 ships a battle-tested `@codemirror/vim` extension that provides modal vim editing. Enabling it as a preferences toggle (following VS Code and Obsidian's pattern) costs negligible bundle size via dynamic import and is highly requested by engineers who are a primary MarkFlow target audience.

## Steps
- Enable vim mode from preferences or the command palette and confirm the editor enters Normal mode with a block cursor.
- Navigate with `hjkl`, enter Insert mode with `i`, and return to Normal with `Escape`.
- Perform a `dd` delete-line, `yy` yank-line, and `p` paste and confirm the document updates correctly without corrupting markdown syntax.

## Automated Verification
- Add integration tests for mode transitions (Normal→Insert→Normal), common motion/operator combinations, and `Escape` behavior that does not conflict with MarkFlow WYSIWYG toggle.

## Manual Verification
- Write and edit a full markdown document using only vim keybindings and confirm no loss of MarkFlow-specific shortcuts (mode toggle, find/replace).
