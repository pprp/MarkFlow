# MF-077 - Clear formatting command removes inline markup from the current selection

## Notes
Implemented on 2026-04-16 in `packages/editor/src/editor/clearFormatting.ts`, `packages/editor/src/editor/extensions/smartInput.ts`, `packages/editor/src/editor/MarkFlowEditor.tsx`, `packages/editor/src/App.tsx`, `packages/shared/src/index.ts`, and `packages/desktop/src/main/menu.ts`. This run adds a selection-aware clear-formatting transform that unwraps or splits supported inline wrappers (bold, italic, inline code, strikethrough, highlight, underline, and links) so the selected text becomes plain text while adjacent unselected text keeps its original formatting. The action is now exposed through `Cmd/Ctrl+\`, the command palette, and the desktop menu bridge. Automated coverage in `packages/editor/src/editor/__tests__/clearFormatting.test.ts`, `packages/editor/src/editor/__tests__/MarkFlowEditor.test.tsx`, `packages/editor/src/__tests__/App.test.tsx`, and `packages/desktop/src/main/menu.test.ts` covers partial-range unwrapping plus shortcut/menu/palette dispatch. On 2026-04-23, `packages/editor/src/editor/__tests__/clearFormatting.test.ts` added a live middle-selection regression for nested bold+italic+link formatting, replacing the remaining manual-only nested-formatting blocker. Sources: https://support.typora.io/Shortcut-Keys/ and https://support.typora.io/Markdown-Reference/.

## Steps
- Select a span containing bold, italic, inline code, and a link.
- Run Clear Formatting (menu, palette, or Cmd/Ctrl+\).
- Confirm the selection becomes plain text while surrounding formatting is untouched.
