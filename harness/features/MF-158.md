# MF-158 - Whitespace and soft line break preferences control editing and export rendering

## Notes
Typora documents explicit behavior for whitespace and line breaks, including soft line breaks, hard line breaks, and optional whitespace-related preferences that affect editing/rendering. MarkFlow has export pipeline coverage (MF-138), preference infrastructure (MF-108), and editor display work (MF-105), but the ledger did not yet track Typora-style controls for making soft line breaks and whitespace rendering/export behavior predictable. Sources: https://support.typora.io/Line-Break/ and https://support.typora.io/Markdown-Reference/.

## Steps
- Open a document containing soft line breaks, hard line breaks, blank lines, and trailing spaces.
- Toggle the whitespace/line-break editing preference and confirm the WYSIWYG editor display changes without mutating markdown source.
- Export HTML/Markdown with the preference enabled and disabled, and confirm soft line breaks and hard line breaks follow the selected behavior.
- Confirm unrelated smart typography and source-mode rendering stay unchanged.

## Automated Verification
- Add editor tests proving whitespace/line-break preferences alter rendered decorations or preview output without changing the document text.
- Add export tests proving configured soft/hard line-break handling is reflected in exported HTML/Markdown.
- Run relevant editor preference, export, lint, build, and `pnpm harness:verify` checks.

## Manual Verification
- In the editor, toggle the whitespace and line-break preferences on a mixed-line-break document and compare WYSIWYG display plus exported output.
