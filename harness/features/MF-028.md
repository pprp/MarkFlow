# MF-028 - Find and replace can search the active document without leaving the editing flow

## Notes
Typora documents in-document search and replace, including case-sensitive, whole-word, and regex options. MarkFlow currently imports `searchKeymap` and `highlightSelectionMatches()` in `packages/editor/src/editor/MarkFlowEditor.tsx`, but it does not expose a verified search/replace workflow in the app or ledger. Sources: https://support.typora.io/Search/ and https://support.typora.io/What's-New-1.3/.

## Steps
- Open a markdown document with repeated terms across headings, prose, and list items.
- Trigger find and replace from the editor and navigate between matches in both source and WYSIWYG mode.
- Replace the current match and all matches, then confirm the underlying markdown updates without losing selection or corrupting surrounding syntax.

## Automated Verification
- Add focused editor tests for search panel open state, match navigation, and replace/replace-all behavior against the live CodeMirror document.

## Manual Verification
- Verify case-sensitive, whole-word, and regex search options behave as expected on a mixed markdown document.
