# MF-123 - Saving an untitled new file prompts for the target folder and filename

## Notes
User-reported issue on 2026-04-21: saving a newly created document has problems. When the active document is untitled, Save should behave like Save As: show the native save dialog, let the user choose the destination folder and filename, then bind the active document/tab to that chosen path.

## Steps
- Create a new untitled document, type content, and trigger File > Save or `Cmd/Ctrl+S`.
- Confirm the native save dialog opens with a sensible default filename and markdown filter.
- Choose a different folder and filename, save, and confirm the active tab/window title reflects the chosen name.
- Reopen the saved file and confirm its contents match the editor buffer.
- Repeat after opening an existing file and then creating a new untitled file; saving the new file must not overwrite or reuse the previously opened path.

## Automated Verification
- Add desktop file-manager coverage proving Save on an untitled document invokes `showSaveDialog`, writes the selected path, emits `file-saved`, and records the chosen path.
- Add renderer coverage proving Save routes the active untitled tab content and tab id to the desktop save handler.

## Manual Verification
- In the desktop app, create a new file from an existing saved-document session, press Save, choose a new folder/name, and verify the original file remains unchanged.
