# MF-056 - Go-to-line dialog jumps the caret and viewport to an arbitrary line number instantly

## Notes
Implemented on 2026-04-12 via a lightweight `packages/editor/src/components/GoToLine.tsx` overlay opened from `packages/editor/src/App.tsx` with `Cmd/Ctrl+L`. The app computes the target document offset from the requested line number, clamps out-of-range input to the document bounds, and reuses the existing `MarkFlowEditor` navigation request path so `EditorView.dispatch` sets the caret and runs `scrollIntoView` without recreating the editor. Automated coverage lives in `packages/editor/src/__tests__/App.test.tsx` and passed on 2026-04-12 with `pnpm --filter @markflow/editor test:run -- --grep go-to-line`, plus `pnpm --filter @markflow/editor build` and `pnpm --filter @markflow/editor lint`. `passes` remains false only because the required manual large-document shortcut smoke test is still pending.

## Steps
- Press Cmd/Ctrl+L, enter a line number, and confirm the caret moves to that line.
- Confirm the viewport scrolls so the target line is visible near the top of the editor.
- Test with a line number beyond the document end and confirm a graceful clamp.
