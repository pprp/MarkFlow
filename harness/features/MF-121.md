# MF-121 - Central command registry backs the command palette, menus, and custom keybindings

## Notes
Commands are currently registered in multiple places (palette, menu, keymap). Unifying them is the prerequisite for a usable keybinding editor and for surfacing per-command help.

## Steps
- Extend `packages/editor/src/components/commandPaletteRegistry.ts` into a single command registry that owns id, title, default keybinding, and handler for every editor and desktop action.
- Refactor the command palette, desktop menu builder, and keymap extension to read from the registry.
- Expose the registry to the custom keybinding editor (MF-096) so user overrides live alongside defaults.
