# MF-144 - Math and diagram renderers load only when visible content requires them

## Notes
The editor currently supports KaTeX math and Mermaid-style diagram previews, but those capabilities should stay off the startup path for ordinary prose documents. This feature keeps math and diagram rendering demand-driven: detect visible math or diagram blocks, load the renderer chunk only when needed, cache successful output, and fall back to source-safe placeholders when loading or rendering fails.

## Steps
- Open a plain markdown document with no math or diagram fences and confirm no KaTeX or Mermaid renderer chunk is requested during startup.
- Open a document containing math and Mermaid blocks below the fold and confirm renderer chunks load only when those blocks enter the decoration viewport window.
- Scroll away and back to previously rendered blocks and confirm cached output is reused without mutating the fenced or inline markdown source.
- Trigger a renderer load or parse failure and confirm the editor keeps the original source editable with a clear inline error state.

## Automated Verification
- Add editor tests proving plain documents do not initialize math or diagram renderers.
- Add decoration tests for viewport-triggered renderer loading, cache reuse, and source-safe failure fallback.
- Add a build-size assertion or bundle report check showing the initial editor entry does not include KaTeX or Mermaid renderer code.

## Manual Verification
- Build the editor, open DevTools Network in a fresh session, load a prose-only document, and confirm math/diagram chunks are absent until a math or diagram block becomes visible.
