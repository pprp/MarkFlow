# MF-116 - Markdown parser receives property-based fuzz coverage for strict and tolerant modes

## Notes
Strict vs. tolerant parsing (MF-100) is a common source of subtle bugs. Property-based coverage catches regressions that example-based tests miss.

## Steps
- Add a `fast-check` (or equivalent) property-based test suite that generates heading, list, and emphasis variants and compares strict vs. tolerant mode outcomes.
- Ensure the suite asserts that no generated input crashes the parser or produces a decoration range outside the document.
- Wire the suite into `pnpm test` without materially increasing CI runtime.
