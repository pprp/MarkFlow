# MF-088 - Document tabs keep multiple markdown files open in a single window

## Notes
Typora's later 1.x multi-tab workflow includes switching between opened documents and reopening the last closed file from the keyboard, not just showing multiple tabs. Repo search found no MarkFlow tab/session model, no keyboard tab-cycling bindings, and no reopen-closed stack in the current desktop/editor code, so the existing tab entry needed those acceptance criteria spelled out. Sources: https://support.typora.io/What%27s-New-1.5/ and https://support.typora.io/Shortcut-Keys/. Implementation now lands per-tab editor snapshots, desktop close-confirm/session IPC, tab cycling/reopen commands, and focused coverage in `packages/editor/src/__tests__/App.test.tsx`, `packages/desktop/src/main/fileManager.test.ts`, and `packages/desktop/src/main/menu.test.ts`; manual desktop verification is still pending, so `passes` remains false.

## Steps
- Open two markdown files and confirm each lives in its own tab with dirty-state indicators.
- Use Typora-style document-switch shortcuts (`Ctrl+Tab` / `Ctrl+Shift+Tab` on Windows/Linux, `Cmd+`` / `Cmd+Shift+`` on macOS) and confirm the active tab changes while caret, selection, outline position, and undo history persist per tab.
- Close a clean tab, trigger `Shift+Cmd/Ctrl+T`, and confirm the most recently closed document reopens in the same window with its prior tab state restored.
- Close a dirty tab and confirm a save/discard/cancel prompt appears and respects the user choice.

## Automated Verification
- Add tab-state tests (open/switch/close/reopen/persist) in `App.test.tsx` and a desktop counterpart covering keyboard tab cycling, reopen-closed history, and window session restore.

## Manual Verification
- Open five tabs, cycle them with the platform shortcuts, reopen the last closed tab, then quit and relaunch to confirm session restore reproduces the tab set.
