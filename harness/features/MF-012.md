# MF-012 - Theme engine hot-reloads CSS themes and lets the user switch themes

## Notes
Completed on 2026-04-10. `packages/desktop/src/main/themeManager.ts` now bootstraps built-in CSS themes into the desktop user-data directory, persists the selected theme, and hot-reloads the active stylesheet through chokidar watchers. `packages/desktop/src/preload/index.ts`, `packages/shared/src/index.ts`, and `packages/editor/src/App.tsx` expose the theme bridge to the renderer so the titlebar theme selector can switch themes and apply CSS overrides without restarting the app. Verified with focused desktop `themeManager.test.ts`, renderer `App.test.tsx`, plus full `pnpm lint`, `pnpm test`, `pnpm build`, `pnpm harness:verify`, and `./harness/init.sh --smoke` runs.

## Steps
- Install or modify a theme stylesheet.
- Trigger the theme watcher.
- Confirm the editor updates its appearance without a full restart.

## Automated Verification
- Add theme watcher integration tests in the desktop package.

## Manual Verification
- Switch between at least two distinct themes and verify persisted selection.
