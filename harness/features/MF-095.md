# MF-095 - Document statistics panel surfaces words, lines, characters, reading time, and per-selection counts

## Notes
Implemented on 2026-04-24 in `packages/editor/src/app-shell/AppStatusBar.tsx` and `packages/editor/src/__tests__/App.test.tsx`. The status-bar summary already surfaced total line counts, but the statistics popover table omitted them; it now renders a `Lines` row for both the document and the active selection. Selection lines are derived from the highlighted source text so they match the same raw line-count semantics as the document summary. UI coverage in `packages/editor/src/__tests__/App.test.tsx` now asserts the document-side `Characters (no spaces)` row, selection-side `Characters` and `Reading time` cells, and selection behavior while the fenced-code exclusion toggle is flipped. Existing `packages/editor/src/editor/__tests__/wordCount.test.ts` coverage continues to verify words, document lines, characters, reading time, selection counts, and fenced-code exclusion behavior in `computeStats`. ## Steps
- Open the statistics panel from the menu or status bar and confirm rows for words, lines, characters, characters without spaces, paragraphs, and approximate reading time.
- Make a multi-block selection and confirm the panel adds a Selection column with line, word, character, paragraph, and reading-time counts for the highlighted text.
- Toggle `Exclude fenced code blocks from counts` and confirm words, characters, and reading time update while the line row remains tied to the raw document/selection text.
