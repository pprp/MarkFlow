# MF-026 - Smart paste converts rich clipboard content into markdown while preserving plain-text paste

## Notes
Typora pastes HTML by converting it to Markdown by default and supports an explicit plain-text or Markdown-source paste path, while MarkFlow currently relies on default browser/CodeMirror paste behavior with no clipboard normalization layer. Sources: https://support.typora.io/Copy-and-Paste/ and https://support.typora.io/Quick-Start/.

## Steps
- Copy rich content from a browser or rich-text editor that includes headings, lists, links, and inline emphasis.
- Paste into MarkFlow in WYSIWYG mode and confirm markdown source is synthesized instead of raw HTML or flattened plain text.
- Use the plain-text paste path and confirm literal text or markdown source is inserted without rich-format conversion.

## Automated Verification
- Add clipboard transformation tests covering HTML-to-markdown conversion and plain-text paste fallback.

## Manual Verification
- Paste from a browser page and a rich editor, then inspect both the rendered result and the underlying markdown.
