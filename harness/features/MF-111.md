# MF-111 - Accessibility audit hardens palette, search, sidebar, and floating toolbar dialogs

## Notes
The dialogs in `packages/editor/src/components/` lack audited ARIA semantics. Adding `jest-axe`/`@axe-core/react` coverage prevents regression as more UI lands.

## Steps
- Add ARIA roles, labels, and focus traps to the command palette, quick open, global search, go-to-line, vault sidebar, and floating toolbar.
- Ensure every interactive control has a visible focus ring and can be reached via keyboard from the editor surface.
- Add automated axe-core smoke tests for the dialog components and confirm no serious/critical violations remain.

## Automated Verification
- pnpm --filter @markflow/editor exec vitest run
- pnpm --filter @markflow/editor lint

## Manual Verification
- Exercise a screen reader (VoiceOver on macOS) through each dialog and confirm it reads labels and state changes.
