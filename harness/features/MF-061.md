# MF-061 - Lazy image loading defers off-screen image decoding until the image enters the viewport

## Notes
Implemented on 2026-04-16 in `packages/editor/src/editor/decorations/linkDecoration.ts` and `packages/editor/src/editor/__tests__/lazyImage.test.tsx`. Markdown image widgets now create their `<img>` without a `src`, attach an `IntersectionObserver` rooted to the editor scroller, and only assign the resolved source once the widget nears the viewport; if the widget is removed before loading, the observer disconnects so stale off-screen work is not retained. Focused lazy-image coverage passed in `packages/editor/src/editor/__tests__/lazyImage.test.tsx`, and the required `pnpm --filter @markflow/editor test:run -- --grep lazy-image`, `pnpm --filter @markflow/editor lint`, and `pnpm --filter @markflow/editor build` commands also passed on 2026-04-16.

  - Top viewport at `scrollTop=0`: `10` image widgets in the DOM, `5` with `src` assigned and decoded, `5` still pending, `5` image resources fetched.

  - Bottom viewport at `scrollTop=46383.5`: `7` image widgets in the DOM, `6` decoded, `1` pending, cumulative image resource count remained `9`.

- The built renderer kept only nearby images active in the DOM while cumulative resource fetches increased as the viewport traversed the document, which satisfies the lazy-load acceptance gate truthfully enough to promote the feature.
