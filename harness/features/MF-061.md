# MF-061 - Lazy image loading defers off-screen image decoding until the image enters the viewport

## Notes
Implemented on 2026-04-16 in `packages/editor/src/editor/decorations/linkDecoration.ts` and `packages/editor/src/editor/__tests__/lazyImage.test.tsx`. Markdown image widgets now create their `<img>` without a `src`, attach an `IntersectionObserver` rooted to the editor scroller, and only assign the resolved source once the widget nears the viewport; if the widget is removed before loading, the observer disconnects so stale off-screen work is not retained. Focused lazy-image coverage passed in `packages/editor/src/editor/__tests__/lazyImage.test.tsx`, and the required `pnpm --filter @markflow/editor test:run -- --grep lazy-image`, `pnpm --filter @markflow/editor lint`, and `pnpm --filter @markflow/editor build` commands also passed on 2026-04-16. `passes` remains false because the required DevTools/CPU-throttled scroll verification could not be truthfully completed from this terminal session.

## Steps
- Open a document with 500 embedded images.
- Confirm only images near the viewport are decoded (check DevTools Network).
- Scroll to the bottom and confirm images load progressively.
