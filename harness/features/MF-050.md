# MF-050 - Background indexer builds a symbol table for headings and anchors without blocking the UI thread

## Notes
Implemented on 2026-04-15 in `packages/editor/src/editor/indexer.ts`, `packages/editor/src/editor/MarkFlowEditor.tsx`, `packages/editor/src/App.tsx`, and `packages/editor/src/editor/outline.ts`. The background indexer now feeds the live outline and internal-anchor navigation through `symbolTableField` plus `onSymbolTableChange`, performs an immediate off-stack initial scan, keeps subsequent rescans debounced, and drops stale async results when documents switch mid-index. On 2026-04-17 the batching path was tightened so it scans line batches incrementally instead of rebuilding prefix-sized symbol tables, uses macrotask scheduling by default to yield between batches, preserves a running scan until a newer debounced restart actually begins, and stops serializing the entire document just to publish symbol-table changes. Automated coverage in `src/editor/__tests__/indexer.test.ts`, `src/editor/__tests__/outline.test.ts`, `src/editor/__tests__/MarkFlowEditor.test.tsx`, and `src/__tests__/App.test.tsx` passed again on 2026-04-17 with the listed editor test command, plus `pnpm --filter @markflow/editor lint`, `pnpm --filter @markflow/editor build`, and `pnpm harness:verify`. The real 180k acceptance gate still fails: a URL-targeted clean-profile Electron CDP probe against `/tmp/mf050-live-180k.md` opened the file successfully, but after `10s` of idle time the outline still contained only `1` item (the document title, not `Section 1`), and three scripted keystrokes near `Paragraph 5` took roughly `1476ms`, `1467ms`, and `1535ms` to appear. `passes` remains false because the large-document typing/no-lag verification is still failing in a real app session.
