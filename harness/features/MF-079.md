# MF-079 - Duplicate line or selection shortcut inserts a copy below without touching the clipboard

## Notes
Verified on 2026-04-23 against the implementation present in `packages/editor/src/editor/extensions/smartInput.ts`, which binds duplication to `Shift-Alt-ArrowDown` so `MF-101` can use Typora's documented `Mod-Shift-D` delete-word shortcut without losing duplicate-line coverage. The command duplicates the covered line block directly below without touching the clipboard, lands a single caret on the duplicate, preserves block selection on duplicated multi-line ranges, isolates the change into one undo step, and now has automated fenced-code regression coverage proving a duplicated full code fence remains two syntactically valid fenced code blocks. Covered by `packages/editor/src/editor/__tests__/MarkFlowEditor.test.tsx`. Sources: https://support.typora.io/Shortcut-Keys/ and the `MF-101` prerequisite shortcut-conflict fix recorded on 2026-04-16.

## Steps
- Place caret on a line and press Shift-Alt-ArrowDown — confirm the line is duplicated directly beneath and the caret lands on the copy.
- Select multiple lines and repeat — confirm the block duplicates as a contiguous copy.
- Confirm undo collapses the duplication into a single atomic step.

## Automated Verification
- Keymap tests cover line and block duplication in `MarkFlowEditor.test.tsx`.
- A focused regression duplicates an entire fenced code block with `Shift-Alt-ArrowDown`, confirms both copies parse as `FencedCode`, preserves the duplicated block selection, and verifies clipboard write APIs are not called.
