# MF-068 - Table editing hotkeys support row/column insertion and row movement without raw markdown surgery

## Notes
Updated on 2026-04-16: `packages/editor/src/editor/extensions/tableCommands.ts` now adds a bounded WYSIWYG table transaction layer wired through `packages/editor/src/editor/MarkFlowEditor.tsx`, covering row insert below via `Mod-Enter`, last-cell `Tab` row append, row delete via `Mod-Shift-Backspace`, row move up/down via `Alt-Arrow`, and user-reachable column insert/delete hotkeys via `Mod-Alt-ArrowLeft`, `Mod-Alt-ArrowRight`, and `Mod-Alt-Backspace`. Focused coverage in `packages/editor/src/editor/__tests__/tableCommands.test.ts` plus a regression pass in `packages/editor/src/editor/__tests__/MarkFlowEditor.test.tsx` passed on 2026-04-16, and the table rewrites keep pipe rows rectangular with alignment markers intact while header/separator `Alt-Arrow` and non-terminal `Tab` no longer fall through to generic line movement or indentation. `passes` remains false because the large-table/manual acceptance is still pending and one Typora parity gap remains open: redo after some table edits does not yet keep the caret in the affected row/cell (most visibly after inserting a brand-new empty row), so the step requiring selection to stay in the affected cell/row across every undo/redo cycle is not fully closed. Sources: https://support.typora.io/Table-Editing/ and https://support.typora.io/Shortcut-Keys/.

Update 2026-04-19: `packages/editor/src/editor/extensions/tableCommands.ts` now ignores the leading table pipe when resolving the active cell and carries the intended selection through redo so row insert/delete/move and column insert/delete stay on the affected row or cell. Focused automation in `packages/editor/src/editor/__tests__/tableCommands.test.ts` passed on 2026-04-19 via `pnpm --filter @markflow/editor exec vitest run src/editor/__tests__/tableCommands.test.ts`, alongside `pnpm --filter @markflow/editor lint` and `pnpm harness:verify`.

Manual verification completed on 2026-04-19 against the live renderer preview at `http://localhost:5173` with a 20-row, 5-column mixed-alignment table fixture:
- `Cmd+Enter` from `R10C3` inserted a blank row directly below row 10; the table stayed rectangular at `21` body rows and the current selection remained on the inserted row through undo, redo, and Source/Preview toggles.
- `Tab` from `R20C5` appended a new blank row 21; undo restored the caret to `R20C5`, redo restored the appended row, and Source/Preview toggles preserved both the row count and selection.
- `Cmd+Shift+Backspace` on `R12C2` removed row 12, moved selection to `R13C2`, and kept the markdown rectangular through undo, redo, and Source/Preview toggles.
- `Alt+ArrowUp` and `Alt+ArrowDown` moved row 10 up and back down without disturbing the header or delimiter rows; the selected cell stayed on `R10C2` throughout both undo/redo cycles and after toggling Source/Preview.
- `Cmd+Alt+ArrowRight` inserted a blank column to the right of `R08C4`, and `Cmd+Alt+Backspace` removed it again; the table stayed rectangular at `6` then `5` columns, and the selection stayed on the affected inserted/deleted column position through undo, redo, and Source/Preview toggles.
- The delimiter row normalized dash widths to match the widest content after edits, but alignment markers (`:---`, `---:`, `:---:`) stayed intact, so there was no separator drift or malformed markdown output.

`pnpm desktop` still cannot launch the Electron shell cleanly in this workspace because unrelated desktop theme API watcher errors remain in `packages/desktop/src/main/index.ts` and `packages/desktop/src/main/menu.ts`. That did not block truthful acceptance for `MF-068`, because the behavior under test lives in `@markflow/editor` and the live renderer preview exercised the same CodeMirror command path shipped inside the desktop shell. With that live proof plus the automated verification above, `harness/feature-ledger.json` can move to `status=verified`, `passes=true`, and `lastVerifiedAt=2026-04-19T11:32:08Z`.

## Steps
- Place the caret in a rendered markdown table and trigger Typora-style row insertion: `Ctrl`/`Cmd`+`Enter` inserts an empty row below, and pressing `Tab` from the last cell appends a new row.
- Delete the current row with `Ctrl`/`Cmd`+`Shift`+`Backspace`, then use a table command surface to insert/delete a column and move the current row up/down; confirm the pipe table stays rectangular and alignment markers remain intact.
- Undo and redo each table command and confirm selection stays in the affected cell/row without exposing malformed markdown.
