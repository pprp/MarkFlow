# MF-068 - Table editing hotkeys support row/column insertion and row movement without raw markdown surgery

## Notes
Updated on 2026-04-16: `packages/editor/src/editor/extensions/tableCommands.ts` now adds a bounded WYSIWYG table transaction layer wired through `packages/editor/src/editor/MarkFlowEditor.tsx`, covering row insert below via `Mod-Enter`, last-cell `Tab` row append, row delete via `Mod-Shift-Backspace`, row move up/down via `Alt-Arrow`, and user-reachable column insert/delete hotkeys via `Mod-Alt-ArrowLeft`, `Mod-Alt-ArrowRight`, and `Mod-Alt-Backspace`. Focused coverage in `packages/editor/src/editor/__tests__/tableCommands.test.ts` plus a regression pass in `packages/editor/src/editor/__tests__/MarkFlowEditor.test.tsx` passed on 2026-04-16, and the table rewrites keep pipe rows rectangular with alignment markers intact while header/separator `Alt-Arrow` and non-terminal `Tab` no longer fall through to generic line movement or indentation. `passes` remains false because the large-table/manual acceptance is still pending and one Typora parity gap remains open: redo after some table edits does not yet keep the caret in the affected row/cell (most visibly after inserting a brand-new empty row), so the step requiring selection to stay in the affected cell/row across every undo/redo cycle is not fully closed. Sources: https://support.typora.io/Table-Editing/ and https://support.typora.io/Shortcut-Keys/.

## Steps
- Place the caret in a rendered markdown table and trigger Typora-style row insertion: `Ctrl`/`Cmd`+`Enter` inserts an empty row below, and pressing `Tab` from the last cell appends a new row.
- Delete the current row with `Ctrl`/`Cmd`+`Shift`+`Backspace`, then use a table command surface to insert/delete a column and move the current row up/down; confirm the pipe table stays rectangular and alignment markers remain intact.
- Undo and redo each table command and confirm selection stays in the affected cell/row without exposing malformed markdown.
