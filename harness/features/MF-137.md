# MF-137 - Pandoc import converts DOCX, RTF, EPUB, OPML, and other supported formats into editable markdown

## Notes
Typora official Pandoc docs list import support for `.docx`, `.latex`, `.tex`, `.ltx`, `.rst`, `.rest`, `.org`, `.wiki`, `.dokuwiki`, `.textile`, `.opml`, and `.epub`. MarkFlow currently tracks Pandoc export but not import.

Sources: https://support.typora.io/Install-and-Use-Pandoc/ and https://support.typora.io/Export/

## Steps
- Add an Import menu/bridge action that prompts for a supported non-markdown file.
- Invoke configured Pandoc to convert the selected file into markdown and open the converted buffer as an unsaved document.
- Surface missing-Pandoc, unsupported-extension, and conversion-failure errors without replacing the active document.

## Automated Verification
- Unit-test FileManager import command construction for DOCX and OPML inputs.
- Mock Pandoc success and verify the renderer receives converted markdown plus an untitled/dirty state.
- Mock Pandoc failure and verify an error dialog appears while the current editor buffer is unchanged.

## Manual Verification
- Import a DOCX and an OPML file through the desktop menu and confirm the converted markdown opens as an unsaved editable document.
- Repeat with Pandoc unavailable and confirm the current document is unchanged while a visible error appears.
