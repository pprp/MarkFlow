# MF-151 - Backlinks panel surfaces inbound wikilinks and missing note references across the active vault

## Notes
Wikilinks are already supported, and backlinks are the natural next layer for networked notes. This feature adds a lightweight backlinks panel that scans the active vault for inbound `[[page]]` references, shows unresolved references, and lets users navigate without turning MarkFlow into a heavy graph database.

## Steps
- Index wikilink references across the opened vault using existing search and symbol-table infrastructure where possible.
- Show inbound links for the active document with source file, heading context, and snippet text.
- Show missing note references separately so users can create or clean them up.
- Keep backlinks updated after file rename, move, save, and active-document changes.

## Automated Verification
- Add vault indexing tests for inbound links, aliases, missing targets, and renamed files.
- Add App or component tests for the backlinks panel empty, loaded, and stale-refresh states.
- Add navigation tests proving a backlink result opens the source file at the referenced location.

## Manual Verification
- Create a small vault with three linked notes, rename one target, and confirm inbound links plus missing references update correctly.
