# MF-073 - Emoji shortcode autocomplete expands `:name:` into Unicode glyphs

## Notes
Implemented on 2026-04-16 with a small CodeMirror autocomplete source in `packages/editor/src/editor/extensions/emojiAutocomplete.ts`, wired into `packages/editor/src/editor/MarkFlowEditor.tsx`. This run ships a curated starter set of common Typora-style emoji shortcodes: typing `:` plus a known prefix now offers matching completions with glyph previews via completion detail text, accepting a suggestion inserts the Unicode glyph itself, and unknown shortcodes return no completions so the popup dismisses without changing the document. Focused regression coverage in `packages/editor/src/editor/__tests__/emojiAutocomplete.test.ts` passed with the listed Vitest command, and targeted lint, `pnpm --filter @markflow/editor build`, plus `pnpm harness:verify` also passed. `passes` remains false because the required manual caret-position check in live prose is still pending. Sources: https://support.typora.io/Markdown-Reference/ and https://support.typora.io/Shortcut-Keys/.

## Steps
- Type `:smi` and confirm a completion popup lists matching emoji shortcodes with glyph previews.
- Accept a suggestion and confirm the inserted text is the Unicode glyph (not the shortcode) while source mode still preserves user edits.
- Type an unknown shortcode and confirm the completion dismisses gracefully without mutating the document.
