# MF-073 - Emoji shortcode autocomplete expands `:name:` into Unicode glyphs

## Notes
Implemented on 2026-04-16 with a small CodeMirror autocomplete source in `packages/editor/src/editor/extensions/emojiAutocomplete.ts`, wired into `packages/editor/src/editor/MarkFlowEditor.tsx`. This run ships a curated starter set of common Typora-style emoji shortcodes: typing `:` plus a known prefix now offers matching completions with glyph previews via completion detail text, accepting a suggestion inserts the Unicode glyph itself, and unknown shortcodes return no completions so the popup dismisses without changing the document. Focused regression coverage in `packages/editor/src/editor/__tests__/emojiAutocomplete.test.ts` passed with the listed Vitest command, and targeted lint, `pnpm --filter @markflow/editor build`, plus `pnpm harness:verify` also passed.

Verified on 2026-04-22 by adding automated coverage for the prior manual caret-position gate: accepting `:rocket:` in the middle of prose now asserts the document becomes `Hello 🚀 world` and the caret lands immediately after the inserted emoji before the trailing prose, and the MarkFlowEditor wiring test now accepts `:tad` before trailing prose and checks the same caret invariant through the real editor component. The new regressions passed against the existing implementation, so no production code change was needed. MF-073 is promoted because all listed behavior is now covered by automated verification. Sources: https://support.typora.io/Markdown-Reference/ and https://support.typora.io/Shortcut-Keys/.

## Steps
- Type `:smi` and confirm a completion popup lists matching emoji shortcodes with glyph previews.
- Accept a suggestion and confirm the inserted text is the Unicode glyph (not the shortcode) while source mode still preserves user edits.
- Type an unknown shortcode and confirm the completion dismisses gracefully without mutating the document.
