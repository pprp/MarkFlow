# MF-104 - System dark mode applies separate light and dark themes automatically

## Notes
Typora documents separate light and dark theme behavior that follows system appearance automatically. This run extends `packages/desktop/src/main/themeManager.ts`, `packages/desktop/src/preload/index.ts`, `packages/shared/src/index.ts`, and `packages/editor/src/App.tsx` so MarkFlow persists independent light/dark theme ids, exposes them to the renderer, and updates the active theme on Electron `nativeTheme` changes; focused desktop and renderer tests were added, but manual OS-level verification is still pending so `passes` remains false. Sources: https://support.typora.io/Dark-Mode/ and https://support.typora.io/.

2026-04-22T20:45:34+0800 implementation slice: the dark-mode CodeMirror syntax-highlight contrast bug documented on 2026-04-16 is now covered and fixed. `packages/editor/src/editor/syntaxHighlighting.ts` replaces CodeMirror's default light-only highlight palette with a MarkFlow highlight style that emits CSS variables, `packages/editor/src/styles/global.css` defines separate light/dark token variables, and `packages/editor/src/editor/__tests__/syntaxHighlighting.test.tsx` verifies both generated token styles and WCAG AA contrast for the dark syntax palette. `MF-104` remains `status=ready`, `passes=false`, and `lastVerifiedAt=null` until the live desktop OS appearance-switch matrix below is completed.

## Steps
- Choose different light and dark themes from the app theme controls and relaunch MarkFlow; confirm both theme mappings persist.
- While MarkFlow is running, switch the OS appearance between light and dark and confirm the matching theme applies immediately without restart.
- Return to each appearance mode and confirm the previously selected light and dark theme ids are still preserved.
