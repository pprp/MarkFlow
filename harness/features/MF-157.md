# MF-157 - Code fence tools expose line numbers, wrapping, default language, and code actions

## Notes
Typora documents code fence editing preferences and tools beyond basic rendering: line numbers, line wrapping, a default code language, code-block styles, and copy/run-adjacent actions for fenced blocks. MarkFlow already renders fenced code safely (MF-003), syntax-highlights common languages (MF-030), and has copy-focused command coverage (MF-075), but the editor does not yet expose Typora-style per-fence tool controls or code-block display preferences. Sources: https://support.typora.io/Code-Fences/, https://support.typora.io/Code-Fences-Language-Support/, and https://support.typora.io/Code-Block-Styles/.

## Steps
- Open a document with multiple fenced code blocks, including one without a language identifier.
- Enable code fence line numbers and confirm rendered WYSIWYG code blocks show stable line numbers without mutating markdown source.
- Toggle code wrapping and confirm long code lines wrap or scroll according to the preference.
- Configure a default code language and confirm newly inserted empty fences receive that language.
- Use code block actions such as copy code from a rendered block without selecting fence markers.

## Automated Verification
- Add editor tests proving code fence preferences affect rendered block classes or widgets without modifying existing markdown source.
- Add command tests proving new fences can receive the configured default language and copy-code actions copy only code content.
- Run relevant code block decoration, command palette, editor lint, and `pnpm harness:verify` checks.

## Manual Verification
- In the editor, toggle line numbers and wrapping, insert a new code fence, and use the rendered code action controls in a multi-block document.
