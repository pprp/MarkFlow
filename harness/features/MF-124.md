# MF-124 - Split view preview pane always shows raw `#` on the first heading

## Notes
In split-view mode the preview (right) pane is a read-only CodeMirror editor whose default cursor sits at position 0. The WYSIWYG heading decoration hides the `#` marker only when the cursor is *not* on that heading's line. Because position 0 is always inside the first heading's line range, `cursorInside` evaluates to `true` and the `#` is never hidden — producing `# MarkFlow` instead of the rendered heading style.

All subsequent headings render correctly because the cursor at 0 is outside their line range.

Root cause: `inlineDecorations.ts` line 183 — the cursor-inside guard should be skipped for non-editable (split-preview) editors.

## Steps
- Open the default starter document.
- Switch to Split view.
- Observe the first heading in the right (preview) pane — it shows `# MarkFlow` with the hash character visible.
- Click into any line that is not heading 1; the `#` remains visible regardless.
- All other headings (`##`, `###`, etc.) correctly hide their markers.
