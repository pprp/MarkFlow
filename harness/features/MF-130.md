# MF-130 - Status bar shows stale cursor line number after tab is loaded from a snapshot

## Notes
Verified on 2026-04-22. The `useEffect` that initializes `activeCursorLineNumber` now recalculates from the active tab content and cursor position whenever restored or large-file navigation state changes, so the status bar no longer waits for a later editor cursor callback.

Original root cause: the effect computed from `activeTab.content` and `activeTab.cursorPosition`, but only depended on `activeTab?.filePath` and `activeTab?.id`.

## Steps
- Restore or navigate a large-file tab to a non-line-1 cursor position.
- Confirm the status bar immediately shows the restored logical line number.
- Confirm later cursor movement still updates the status bar normally.
