# MF-130 - Status bar shows stale cursor line number after tab is loaded from a snapshot

## Notes
The `useEffect` that initialises `activeCursorLineNumber` (App.tsx line 367–376) runs only when `activeTab?.filePath` or `activeTab?.id` changes. If a tab is restored from a snapshot with the cursor at line 50, the status bar shows "Ln 1" until the user moves the cursor because `activeTab.cursorPosition` is not in the dependency array.

Root cause: `App.tsx` line 376 — dependency array should include `activeTab?.cursorPosition`.

## Steps
- Open a file, scroll to line 50, and move the cursor there.
- Close and reopen the file (restoring from snapshot).
- Observe the status bar shows "Ln 1" even though the cursor is at line 50.
- Move the cursor — the line number then updates correctly.

## Automated Verification
- Unit test: restoring a tab snapshot with `cursorPosition` pointing to line 50 should immediately render "Ln 50" in the status bar without any further cursor movement.

## Manual Verification
- Close and reopen a previously edited file; the status bar line number should match the restored cursor position.
