# MF-048 - Virtual rendering limits DOM nodes to a viewport window for files over 5 000 lines

## Notes
Verified on 2026-04-12. Virtual rendering via `viewportWindow.ts` keeps docs over 5,000 lines bounded to visible ranges + 200-line buffer. Link, TOC, and footnote decorations cache metadata on `docChanged` to avoid full rescans on scroll/selection. Window clamped at both document edges. 231 tests pass including virtualRendering.test.ts. Not-tested: manual 180k-line fixture scroll FPS check.

## Steps
- Open a markdown file with 180 000+ lines.
- Confirm the editor loads in under 3 seconds and the DOM contains fewer than 2 000 rendered lines at any scroll position.
- Scroll rapidly to the bottom and confirm no visible jank or blank regions.

## Automated Verification
- pnpm --filter @markflow/editor test:run -- --grep virtual

## Manual Verification
- Run `pnpm harness:fixture:large`, load `harness/fixtures/mf-large-180k.md`, and observe scroll FPS in DevTools Performance tab.
