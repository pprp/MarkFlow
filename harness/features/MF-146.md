# MF-146 - KaTeX assets ship as a minimal WOFF2 font set loaded only for math documents

## Notes
The current editor build emits multiple KaTeX font formats for the same families. Modern Chromium-based desktop builds can prefer WOFF2, and prose-only documents should not pay the font cost at startup. This feature trims duplicate KaTeX font artifacts and loads math fonts only when math rendering is active.

## Steps
- Configure KaTeX styling so MarkFlow ships and references only the required WOFF2 font assets.
- Ensure math font CSS is loaded only after a visible math expression needs KaTeX rendering.
- Keep math rendering visually correct for inline math, display math, and fenced math blocks.
- Preserve graceful text fallback if font loading fails.

## Automated Verification
- Add build artifact checks proving duplicate KaTeX TTF and WOFF files are not emitted.
- Add editor tests proving math documents request math font CSS while prose-only documents do not.
- Add math decoration regressions to confirm rendered output still uses the KaTeX path.

## Manual Verification
- Build the app, inspect `packages/editor/dist/assets`, and confirm only the intended KaTeX font assets remain; then open a math-heavy document and visually compare inline and display math.
