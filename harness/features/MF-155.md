# MF-155 - Context menu search services open selected text in configured web targets

## Notes
Typora supports configurable search services for selected text from the context menu. MarkFlow already has in-document search and global vault search, but it does not expose a selected-text context-menu workflow that opens configured external web search targets. Source: https://support.typora.io/Add-Search-Service/.

## Steps
- Configure at least one search service with a URL template that accepts selected text.
- Select text in the editor and open the context menu.
- Choose the configured search service and confirm the external target opens with the selected text encoded.
- Confirm empty selections and malformed service URLs do not trigger unexpected navigation.

## Automated Verification
- Add desktop menu or renderer action coverage for selected-text search URL construction.
- Add settings validation coverage for search service URL templates.
- Run `pnpm harness:verify`.

## Manual Verification
- In the desktop app, select text, invoke the context menu search service, and confirm the browser opens the configured search result.
