# MF-054 - Find-and-replace supports regex mode, case-sensitivity toggle, and whole-word matching

## Notes
Typora's Search doc says Cmd/Ctrl+H toggles Find and Replace in the current file and that regex replacement supports `$0`, `$1`, ... capture references, alongside case-sensitive, whole-word, and regexp toggles. MarkFlow now exposes the existing CodeMirror search panel through an explicit `Mod-h` binding in `packages/editor/src/editor/MarkFlowEditor.tsx`, focuses the replace field when opened from the editor, and relies on the built-in panel controls for regex, match-case, whole-word, and replace/replace-all behavior. Focused regression coverage in `packages/editor/src/editor/__tests__/findReplace.test.ts` and `packages/editor/src/editor/__tests__/MarkFlowEditor.test.tsx` now exercises live panel creation, query toggle commits, match navigation, regex capture-group replacement, exact-case replacement, whole-word replacement, and the `Cmd/Ctrl+H` entry point. Automated verification passed on 2026-04-16 with `pnpm --filter @markflow/editor exec vitest run src/editor/__tests__/findReplace.test.ts src/editor/__tests__/MarkFlowEditor.test.tsx` and `pnpm harness:verify`. `passes` remains false because the manual regex-replace fixture check is still pending. Sources: https://support.typora.io/Search/ and https://support.typora.io/What's-New-1.3/.

## Steps
- Press Cmd/Ctrl+H to open the find-and-replace panel from the editor.
- Enable regex mode, enter a pattern, and confirm only regex-matching spans are highlighted.
- Toggle case-sensitivity and confirm the match set changes accordingly.
- Toggle whole-word matching, then use Replace All and confirm all intended occurrences are substituted without corrupting surrounding text.
