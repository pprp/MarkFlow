# MF-156 - Recovery panel lists unsaved draft snapshots and version restore options

## Notes
Typora documents auto-save and version-control style recovery workflows for unsaved drafts and previous file states. MarkFlow has autosave/crash-recovery coverage, but no browsable recovery panel that lists draft snapshots and restore choices for user-controlled recovery. Sources: https://support.typora.io/Auto-Save/ and https://support.typora.io/Version-Control/.

## Steps
- Create or modify a document and trigger an unsaved draft snapshot.
- Open a recovery panel listing available unsaved draft snapshots and previous versions.
- Preview a snapshot or version before restoring.
- Restore a selected snapshot/version and confirm the active document content changes only after confirmation.

## Automated Verification
- Add recovery store coverage for listing snapshots and restoring a selected version.
- Add renderer coverage for the recovery panel empty, populated, preview, and restore states.
- Run `pnpm harness:verify`.

## Manual Verification
- In the desktop app, create a recoverable draft, restart, open the recovery panel, and restore the expected snapshot.
