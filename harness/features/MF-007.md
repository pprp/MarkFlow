# MF-007 - Opening a markdown file from the desktop shell hydrates the editor state and active file path

## Notes
Renderer now hydrates from both `file-opened` events and `getCurrentDocument()` for startup recovery; covered by `packages/editor/src/__tests__/App.test.tsx`.

## Steps
- Launch the Electron app with a markdown file path or use File > Open.
- Confirm the renderer receives the file content and path.
- Verify the editor shows the opened content instead of the starter document.

## Automated Verification
- pnpm --filter @markflow/editor test:run

## Manual Verification
- Open a real `.md` file from Finder or the File menu and verify the buffer updates.
