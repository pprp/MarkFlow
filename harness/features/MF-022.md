# MF-022 - Footnote references render cleanly and reveal editable note content without corrupting markdown

## Notes
Implemented with `packages/editor/src/editor/decorations/footnoteDecoration.ts`, which renders `[^label]` references as superscript markers, derives hover-preview text from matching `[^label]: ...` definitions, and keeps raw markdown visible whenever the caret enters the reference or definition. `packages/editor/src/editor/decorations/linkDecoration.ts` now skips footnote-shaped `Link` nodes so definitions are not misclassified as regular links, and `packages/editor/src/editor/__tests__/footnoteDecoration.test.tsx` covers caret reveal, repeated references, definition-edit round trips, missing definitions, and code/non-footnote false-positive avoidance. Sources: https://support.typora.io/media/export/Markdown-Reference.pdf and https://typora.io/. BUG (found 2026-04-16, detailed live-preview test): Footnote rendering is visually correct in isolation, but when two references appear on the same line (e.g. the starter document's `This has a footnote[^1] and another[^2].`), the superscript regex in `inlineDecorations.ts` pairs the `^` of the first reference with the `^` of the second and wrongly renders the intervening ` and another` as a superscript span. Root cause and fix belong to MF-037, but this regresses the "footnote references render cleanly" guarantee in the mainline starter document; cross-reference added here so the failure is not lost if MF-037 is deferred.

## Steps
- Open markdown containing inline footnote references and matching footnote definitions.
- Render the document in WYSIWYG mode and confirm references appear as superscripts or equivalent footnote markers.
- Hover or otherwise reveal the referenced footnote content, edit the note text, and confirm the underlying markdown references and definitions stay intact.
