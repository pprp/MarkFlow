# MF-119 - Repo root cleanup removes stray one-shot scripts and legacy artifacts

## Notes
Stray files at the repo root make onboarding noisier and accidentally ship generated artifacts into PR diffs. Pure hygiene.

Implemented on 2026-04-22 by deleting the unused root-level one-shot repair scripts (`fix_code_ranges.py`, `fix_footnote.py`, `fix_syntax_tree.py`, `fix_virtual.py`), the empty stray `Bn` file, the legacy Codex loop helper (`run_codex.sh`), and the obsolete standalone `build.sh` wrapper. The canonical build, desktop packaging, CI smoke, and harness flows remain under `package.json`, `scripts/build/`, `scripts/ci/`, `scripts/harness/`, and `harness/init.sh`. The generated artifact ignore rules already covered `dist-mac/`, `dist-win/`, `dist-linux/`, and `dist-desktop/`, so no `.gitignore` change was needed.

## Steps
- Audit the repository root (`fix_code_ranges.py`, `fix_footnote.py`, `fix_syntax_tree.py`, `fix_virtual.py`, `run_codex.sh`, `Bn/`, `dist-mac/`) and either move each into `scripts/` or delete it.
- Update `.gitignore` so generated artifacts like `dist-mac/` are no longer tracked.
- Confirm `pnpm build`, `pnpm test`, and `pnpm harness:verify` all still succeed after the cleanup.
