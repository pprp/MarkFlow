# MF-083 - Reveal in Finder / Show in folder action opens the active file's location

## Notes
Implemented on 2026-04-15 in `packages/desktop/src/main/fileManager.ts`, `packages/desktop/src/main/index.ts`, `packages/desktop/src/main/menu.ts`, `packages/desktop/src/main/fileManager.test.ts`, and `packages/desktop/src/main/menu.test.ts`. The desktop File menu now exposes `Reveal in Finder` on macOS, `Show in File Explorer` on Windows, and `Show in Folder` elsewhere; it stays disabled until the current document has a saved path, rebuilds after open/new/Save As path transitions, and reveals the latest saved path through `shell.showItemInFolder`. Automated verification passed on 2026-04-15 with `pnpm --filter @markflow/desktop exec vitest run src/main/fileManager.test.ts src/main/menu.test.ts`, `pnpm --filter @markflow/desktop lint`, `pnpm --filter @markflow/desktop build`, and `pnpm harness:verify`. `passes` remains false because the required manual cross-platform OS file-manager verification is still pending. Sources: https://support.typora.io/File-Management/ and https://support.typora.io/Shortcut-Keys/.

## Steps
- Open a saved markdown file.
- Trigger the Reveal in Finder / Show in folder menu or palette action.
- Confirm the OS file manager opens with the file highlighted; confirm the action is disabled for untitled/unsaved documents.
