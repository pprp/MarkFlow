# MF-139 - File sidebar supports Typora-style refresh, sort, move, duplicate, copy path, and undo file operations

## Notes
Typora file management docs describe sidebar refresh, folder switching, sort modes, move, duplicate, copy file path, reveal, and undo file operations. MarkFlow already tracks sidebar open/rename/delete, reveal, and recent/pinned locations, but not these remaining file-management actions.

Source: https://support.typora.io/File-Management/

## Steps
- Extend vault sidebar actions for refresh, sort mode, duplicate, move, copy path, and undo when a file operation is reversible.
- Persist the selected sort mode and group-by-folder behavior per opened folder.
- Keep open tabs, recent entries, and represented file paths synchronized after move or duplicate operations.

## Automated Verification
- Component tests cover visible sidebar actions and sort/group mode changes.
- FileManager tests cover duplicate, move, copy-path payloads, refresh, and undo operation bookkeeping.
- App integration tests cover an open tab following a moved or duplicated file path.

## Manual Verification
- In Electron, move an open markdown file from the sidebar, verify the tab follows the new path, then undo the move.
