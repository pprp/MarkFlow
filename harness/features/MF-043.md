# MF-043 - Reading mode renders the document as fully static HTML without an editable code surface

## Notes
MarkFlow's current WYSIWYG mode still reveals raw syntax when the caret is inside a decorated range. Reading mode removes the caret entirely and applies all decorations unconditionally, giving users a clean finished-document view for proofreading and sharing. This is the third point in the source ↔ WYSIWYG ↔ reading triad that Typora expresses as its signature UX.

## Steps
- Toggle reading mode from the toolbar or shortcut and confirm the editor surface becomes non-editable and all markdown syntax is replaced by styled HTML.
- Confirm all WYSIWYG decorations (headings, math, diagrams, callouts, images) render in reading mode without the cursor-reveal behavior.
- Return to editing mode and confirm the document state, selection, and undo history are preserved exactly.

## Automated Verification
- Add tests that verify the reading-mode extension disables all input handling, applies all decoration sets unconditionally, and that toggling back to edit mode restores a live CodeMirror state.

## Manual Verification
- Switch to reading mode on a document containing math, a mermaid diagram, a table, and an admonition block; confirm each renders without any syntax marker visible.
