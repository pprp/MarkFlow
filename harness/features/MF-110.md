# MF-110 - Feature ledger storage is normalized so long-form notes live in per-feature files

## Notes
Implemented on 2026-04-17 in `scripts/harness/shared.mjs`, `scripts/harness/verify.mjs`, `harness/feature-ledger.json`, `harness/features/*.md`, and `harness/README.md`. The harness now stores only feature metadata plus `notesRef` in `harness/feature-ledger.json`, resolves long-form notes/steps/verification from `harness/features/MF-XXX.md`, and validates that each feature markdown file keeps the required `Notes`, `Steps`, `Automated Verification`, and `Manual Verification` sections. This cut the JSON ledger from `210,551` bytes to `48,425` bytes while keeping `pnpm harness:start`, `pnpm harness:next`, and `pnpm harness:verify` output-compatible for the rest of the workflow.

## Steps
- Move long-form `notes`, `steps`, and `verification` prose out of the monolithic `harness/feature-ledger.json` into `harness/features/MF-XXX.md` files.
- Keep only metadata (id, title, status, passes, priority, dependsOn, lastVerifiedAt, area, category, notesRef) in the JSON.
- Update `scripts/harness/*` and `pnpm harness:start` / `pnpm harness:verify` to resolve the split structure, and confirm verify still passes.
