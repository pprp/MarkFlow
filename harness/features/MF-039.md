# MF-039 - Spell checking underlines misspelled words in prose and offers inline correction suggestions

## Notes
Spell checking is expected by writers and academics. In Electron the browser's native `spellcheck` attribute can be used on the contenteditable layer, or a CodeMirror linter powered by `nspell` or a native OS API can provide range-aware suggestions while excluding code and metadata regions.

## Steps
- Type a deliberate misspelling in a prose paragraph and confirm a red squiggly underline appears beneath the word.
- Right-click or invoke the correction shortcut and confirm a list of candidate corrections is offered.
- Verify that words inside fenced code blocks, inline code, URLs, and YAML front matter are excluded from spell checking.

## Automated Verification
- Add unit tests for spell-check range exclusion (code, URL, front matter) to prevent false positives in non-prose regions.

## Manual Verification
- Write a document mixing prose, code, and a URL, confirm only prose words receive squiggle decorations, and accept one correction suggestion.
