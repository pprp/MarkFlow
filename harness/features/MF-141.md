# MF-141 - Export and print honor Typora-style manual and heading-based page breaks

## Notes
Typora documents print/export pagination through CSS-driven automatic page breaks before headings, forced page-break HTML blocks, and thematic-break-based print rules. MarkFlow already has HTML/PDF export quality work, but the ledger did not track explicit author-controlled page breaks or heading-based pagination parity.

Sources: https://support.typora.io/Page-Breaks/ and https://support.typora.io/Export/

## Steps
- Preserve raw HTML page-break markers such as `<div class="page-break"></div>` or inline `page-break-after` / `break-after` styles through the export sanitizer and renderer.
- Add export print CSS support for optional heading-based page breaks and thematic-break-driven page breaks without affecting normal editor rendering.
- Ensure HTML and PDF export use the same pagination rules so repeated exports are predictable.

## Automated Verification
- Add HTML export tests proving page-break elements and safe break styles survive export output.
- Add PDF/export plumbing tests proving print CSS includes the configured page-break rules when enabled.
- Verify normal editor rendering and non-print HTML output do not visually expose page-break controls outside print/export contexts.

## Manual Verification
- Export a document with manual page-break HTML before a section and confirm the generated PDF starts that section on a new page.
- Enable heading-based page breaks, export a multi-H1 document, and confirm each top-level heading starts on a new page except the first when configured.
