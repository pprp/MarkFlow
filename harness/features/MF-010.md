# MF-010 - Rendered links are clickable and open externally from the desktop shell

## Notes
Typora documents `Cmd`/`Ctrl`+click behavior plus external, internal, local, reference, and auto URL link handling. Sources: https://support.typora.io/Links/ and https://support.typora.io/Markdown-Reference/. Completed on 2026-04-10: MarkFlow now renders inline links, reference-style links, bare URLs, and autolinks as clickable anchors in `packages/editor/src/editor/decorations/linkDecoration.ts`; `packages/editor/src/editor/MarkFlowEditor.tsx` routes Cmd/Ctrl+click internal anchors to matching headings and local markdown links through the desktop bridge; and the Electron shell blocks app-local `window.open` calls while keeping external URLs delegated to the system browser. Verified with targeted editor/desktop Vitest coverage plus full `pnpm lint`, `pnpm test`, `pnpm build`, `pnpm harness:verify`, and `./harness/init.sh --smoke`.

## Steps
- Create markdown with external links, an internal heading link, and at least one reference-style link.
- Render the document in WYSIWYG mode and move the caret away from each link.
- Confirm `Cmd`/`Ctrl`+click opens external URLs in the system browser, internal links jump within the document, and the Electron window never navigates away.
