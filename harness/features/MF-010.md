# MF-010 - Rendered links open external/local targets and guide creation for missing local files

## Notes
Typora documents `Cmd`/`Ctrl`+click behavior plus external, internal, local, reference, and auto URL link handling. For local file links, Typora also guides the user to create a missing target file and continue. Sources: https://support.typora.io/Links/ and https://support.typora.io/File-Management/.

Completed on 2026-04-10: MarkFlow renders inline links, reference-style links, bare URLs, and autolinks as clickable anchors in `packages/editor/src/editor/decorations/linkDecoration.ts`; `packages/editor/src/editor/MarkFlowEditor.tsx` routes Cmd/Ctrl+click internal anchors to matching headings and local markdown links through the desktop bridge; and the Electron shell blocks app-local `window.open` calls while keeping external URLs delegated to the system browser.

Reopened slice implemented on 2026-04-23: missing local markdown links now retry through a typed `openPath(..., { createIfMissing: true })` bridge path after the normal existing-file open fails. `packages/desktop/src/main/fileManager.ts` shows a native "Create and Open" confirmation containing the target path, creates parent directories plus the empty target file on confirmation, and opens the created file through the existing open path. If creation is cancelled or fails, `packages/editor/src/app-shell/useNavigationHistoryController.ts` shows a linked-file-specific toast instead of the previous recent-file message. Existing-file opens still use the original one-argument bridge call, and external links still route through `window.open`.

## Steps
- Create markdown with external links, an internal heading link, a local markdown file link, and a missing local markdown file link.
- Render the document in WYSIWYG mode and move the caret away from each link.
- Confirm `Cmd`/`Ctrl`+click opens external URLs in the system browser, internal links jump within the document, and the Electron window never navigates away.
- Confirm a missing local markdown link offers to create the shown target path, then opens the newly created file after confirmation.
