# MF-106 - Paragraph shortcuts insert table, code fence, and math block scaffolds without raw markdown typing

## Notes
Updated on 2026-04-16: `packages/editor/src/editor/extensions/smartInput.ts` inserts table, fenced code block, and math block markdown scaffolds only when the active selection is a plain paragraph in WYSIWYG mode, using Typora-aligned bindings (`Ctrl+T`, `Ctrl+Shift+K`, `Ctrl+Shift+M` on Windows/Linux; `Cmd+Opt+T`, `Cmd+Opt+C`, `Cmd+Opt+B` on macOS). Automated coverage in `packages/editor/src/editor/__tests__/smartInput.test.ts`, `packages/editor/src/editor/__tests__/smartInput.mac.test.ts`, and `packages/editor/src/editor/__tests__/MarkFlowEditor.test.tsx` now proves the Win/Linux DOM shortcut path, a simulated Mac DOM shortcut path under `navigator.platform = MacIntel`, scaffold text/caret placement, source-mode editability after toggle, and no-op behavior outside WYSIWYG/non-paragraph contexts, so the feature can be truthfully marked verified. Source: https://support.typora.io/Shortcut-Keys/.

## Steps
- Place the caret on a plain paragraph and press `Ctrl+T` on Windows/Linux or `Cmd+Opt+T` on macOS; confirm MarkFlow inserts a valid markdown table scaffold with a header row, separator row, and one body row, then places the caret inside the first editable cell.
- Press `Ctrl+Shift+K` on Windows/Linux or `Cmd+Opt+C` on macOS; confirm a fenced code block scaffold is inserted at the current block and the caret lands inside the fence body.
- Press `Ctrl+Shift+M` on Windows/Linux or `Cmd+Opt+B` on macOS; confirm a `$$` math block scaffold is inserted at the current block and the caret lands inside the math body without corrupting surrounding markdown.
