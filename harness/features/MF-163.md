# MF-163 - Clipboard copy options support semantic HTML and Typora-style default copy behavior preferences

## Notes
Typora documents `Copy without Theme Styling` and a default copy-behavior preference that adapts clipboard output for code editors versus rich-text editors. MarkFlow currently supports default rich copy plus explicit plain-text, Markdown, and HTML-code actions, but it does not yet expose a Typora-style semantic-copy mode or preference-driven default copy behavior. Sources: https://support.typora.io/Copy-and-Paste/

## Steps
- Select rendered content containing emphasis, links, and inline code.
- Run the semantic-copy path and confirm the clipboard exposes clean HTML without theme styling while preserving rendered plaintext.
- Configure the default copy behavior preference and confirm copying into a code editor yields Markdown/plain text while rich-text targets still receive semantic HTML.
- Verify existing `Copy`, `Copy as Plain Text`, `Copy as Markdown`, and `Copy as HTML Code` actions still behave as documented.

## Automated Verification
- Add renderer clipboard tests for semantic HTML output without theme CSS leakage.
- Add preference-driven copy-behavior tests that distinguish code-editor and rich-text targets.
- Run relevant editor/desktop clipboard tests and `pnpm harness:verify`.

## Manual Verification
- Copy the same rendered selection into a code editor and a rich-text app, then confirm the configured default behavior matches Typora's documented clipboard modes.
