# MF-075 - Copy writes rich clipboard formats, while Copy as Markdown and Copy as HTML Code expose source formats

## Notes
Implemented with explicit `copy`, `copy-as-markdown`, and `copy-as-html-code` menu actions, a bounded preload clipboard bridge, and a minimal renderer-side markdown selection serializer for bold, emphasis, links, inline code, and paragraph/line-break structure. Automated coverage now asserts default copy writes `text/html` plus rendered plaintext, while the source-oriented actions write markdown text or HTML fragment text through `packages/editor/src/__tests__/App.test.tsx` and `packages/desktop/src/main/menu.test.ts`. `passes` remains false until a manual cross-app clipboard check confirms real Electron clipboard behavior in both rich-text and plain-text targets. Sources: https://support.typora.io/Copy-and-Paste/ and https://support.typora.io/Shortcut-Keys/.

## Steps
- Select text containing bold, a link, and inline code.
- Run the normal `Copy` action and paste into both a rich-text target and a plain-text target; confirm the default clipboard payload carries rendered rich content.
- Run `Copy as Markdown` and confirm a plain-text paste contains raw markdown source, then run `Copy as HTML Code` and confirm a plain-text paste contains HTML source.

## Automated Verification
- pnpm --filter @markflow/editor exec vitest run src/__tests__/App.test.tsx
- pnpm --filter @markflow/desktop exec vitest run src/main/menu.test.ts
- pnpm --filter @markflow/editor build
- pnpm harness:verify

## Manual Verification
- Paste the same selection into a rich-text app and a plain-text/code editor to confirm all three copy paths behave like Typora.
