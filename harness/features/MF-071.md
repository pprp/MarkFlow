# MF-071 - Spellcheck settings allow language selection and custom dictionary behavior in-editor

## Notes
Typora release notes continue adding UI language support and mature spellcheck workflows typically include configurable dictionaries; MarkFlow currently enables browser spellcheck with exclusion ranges but no exposed language/dictionary controls in editor scope. Sources: https://support.typora.io/What%27s-New-1.13/ and https://support.typora.io/What%27s-New-1.10/.

## Steps
- Open spellcheck settings and choose a non-default language dictionary for the active document or app profile.
- Add a domain-specific term to a custom dictionary and confirm it is no longer flagged as misspelled.
- Switch back to default language and confirm language-specific misspelling detection changes accordingly.

## Automated Verification
- Add tests around spellcheck configuration plumbing (language selection + per-profile dictionary persistence) while preserving existing exclusion ranges.

## Manual Verification
- Toggle between two dictionaries (for example en-US and de-DE) and verify misspelling highlights differ on the same text.
