# MF-102 - Zoom in, zoom out, and reset zoom adjust the window scale without disturbing document state

## Notes
Typora exposes zoom from the View menu plus shortcut-driven reset or actual-size flows. MarkFlow already wires Electron `resetZoom`, `zoomIn`, and `zoomOut` roles in `packages/desktop/src/main/index.ts`, but the capability had no ledger entry, no explicit verification, and no Typora-like zoom state affordance yet. Sources: https://support.typora.io/Zoom/ and https://support.typora.io/Shortcut-Keys/.

## Steps
- Use View > Zoom In / Zoom Out or the matching shortcuts and confirm the window scale changes while the current document content, selection, and scroll position remain intact.
- Trigger Actual Size / Reset Zoom and confirm the scale returns to baseline without recreating the editor view.
- Continue editing after zooming and confirm typing, toolbar actions, and file save behavior still match the unzoomed editor.

## Automated Verification
- Add a desktop test that asserts the View menu exposes `resetZoom`, `zoomIn`, and `zoomOut` roles or equivalent BrowserWindow zoom commands.

## Manual Verification
- Interactive desktop check: zoom in, zoom out, and reset on a live document while watching selection and scroll stability.
