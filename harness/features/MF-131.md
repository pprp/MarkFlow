# MF-131 - Outline toggle button appears in titlebar when sidebar already shows the outline

## Notes
`shouldShowOutlineToggle` (`App.tsx` line 1788) is `true` whenever there are headings and the app is not in immersive mode, regardless of whether the sidebar is open. When the sidebar is visible it already includes the outline section, but the titlebar toggle button still appears, giving the impression of a separate collapsible panel that doesn't exist.

Root cause: `App.tsx` line 1788 — the condition should also be `false` when `showSidebar` is `true`.

## Steps
- Open a document with headings.
- Toggle the file sidebar open.
- Observe the outline toggle button still appears in the top-right of the titlebar.
- Clicking it collapses/expands a standalone outline panel that isn't visible because the sidebar is covering it.

## Automated Verification
- Unit test: when `showSidebar` is `true` and `outlineHeadings.length > 0`, `shouldShowOutlineToggle` should be `false`.

## Manual Verification
- With the sidebar open, the outline toggle button should not appear in the titlebar.
- With the sidebar closed, the outline toggle should appear and function normally.
