# MF-097 - Embedded HTML media tags render preserved `<video>`, `<audio>`, and `<iframe>` content in WYSIWYG mode

## Notes
Typora's current HTML and Media docs explicitly support `<video>`, `<audio>`, and `<iframe>` embeds, including drag-and-drop video insertion and sandboxed iframe/script handling. MarkFlow now preserves those tags in `packages/editor/src/editor/decorations/inlineHtmlDecoration.ts` with a small safe attribute allowlist, iframe sandboxing, and script/event-handler stripping, covered by `packages/editor/src/editor/__tests__/inlineHtmlDecoration.test.ts`. Manual local media/embed playback validation is still pending before this can move to `verified`. Sources: https://support.typora.io/HTML/ and https://support.typora.io/Media/.

## Steps
- Insert raw HTML media tags such as `<video controls src='sample.mp4'></video>`, `<audio controls src='sample.mp3'></audio>`, and an embeddable `<iframe ... allowfullscreen></iframe>` snippet into a markdown document.
- Render in WYSIWYG mode and confirm the media/embed element appears, required safe attributes (`src`, `controls`, `width`/`height`, `allowfullscreen`, etc.) survive, and the underlying markdown source is preserved.
- Move the caret back into the HTML block and confirm the raw source becomes editable again without corrupting surrounding blocks or letting the embed navigate the editor shell.
