# MF-097 - Embedded HTML media tags render preserved `<video>`, `<audio>`, `<track>`, and `<iframe>` content with Typora-style media paths

## Notes
Typora's current HTML and Media docs explicitly support `<video>`, `<audio>`, `<track>`, and `<iframe>` embeds, including drag-and-drop video insertion, media paths relative to the current markdown file, and sandboxed iframe/script handling. MarkFlow now preserves those tags in `packages/editor/src/editor/decorations/inlineHtmlDecoration.ts` with a small safe attribute allowlist, iframe sandboxing, script/event-handler stripping, and relative `src`/`poster` resolution for media elements and nested `<source>` / `<track>` children. Coverage lives in `packages/editor/src/editor/__tests__/inlineHtmlDecoration.test.ts`. Manual local media/embed playback validation is still pending before this can move to `verified`. Sources: https://support.typora.io/HTML/ and https://support.typora.io/Media/.

## Steps
- Insert raw HTML media tags such as `<video controls src='sample.mp4'><track src='captions.vtt' kind='subtitles' srclang='en'></video>`, `<audio controls src='sample.mp3'></audio>`, and an embeddable `<iframe ... allowfullscreen></iframe>` snippet into a markdown document.
- Render in WYSIWYG mode and confirm the media/embed element appears, required safe attributes (`src`, `poster`, `controls`, `width`/`height`, `kind`, `srclang`, `allowfullscreen`, etc.) survive, local media paths resolve relative to the markdown file, unsafe URLs are stripped, and the underlying markdown source is preserved.
- Move the caret back into the HTML block and confirm the raw source becomes editable again without corrupting surrounding blocks or letting the embed navigate the editor shell.
