# MF-128 - Document search match count shows "1 matches" (grammar) and "0 matches" flash

## Notes
Two related issues in `DocumentSearch.tsx`:

1. **Grammar**: The status text `\`${matchCount} matches\`` is always plural. When exactly one result is found, the UI displays "1 matches" instead of "1 match".

2. **0 matches flash**: The match count starts at `0` and is updated asynchronously by a Web Worker. While the worker computes, the UI briefly displays "0 matches" instead of a loading indicator, which looks like a false negative.

Root cause: `DocumentSearch.tsx` line 47.

## Steps
- Press `Ctrl+F` / `Cmd+F` to open document search.
- Type a query that matches exactly one word in the document.
- Observe "1 matches" in the counter.
- Type a query and immediately observe — "0 matches" briefly appears before the real count.

## Automated Verification
- Unit test: assert that `matchCount === 1` renders "1 match" and `matchCount === 2` renders "2 matches".

## Manual Verification
- Search for a term with one result; count reads "1 match".
- Search for a common word; no "0 matches" flash appears while counting.
