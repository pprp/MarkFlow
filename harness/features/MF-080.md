# MF-080 - Auto-pair brackets, quotes, backticks, and markdown delimiters on insertion

## Notes
Completed on 2026-04-15. `packages/editor/src/editor/extensions/smartInput.ts` keeps unconditional pairing for structural delimiters and routes `*`/`_` through a markdown-specific key path: selections still wrap, paired delimiters still support skip-over and empty-pair backspace, but `*` falls through at line start so unordered-list entry still works and `_` falls through inside identifiers/plain text such as `snake_case`. `packages/editor/src/editor/__tests__/smartInput.test.ts` exercises the real keymap path for both the paired and literal-entry cases. Sources: https://support.typora.io/Auto-Pair/ and https://support.typora.io/Markdown-Reference/.

## Steps
- Type `(`, `[`, `{`, `"`, `'`, and `` ` `` and confirm each emits a matching closing character with the caret placed between the pair.
- Type `*` or `_` in an emphasis-friendly text position and confirm the editor inserts a matching closing delimiter, wraps selections with the typed delimiter, and Backspace removes an empty pair atomically.
- Type `*` at the start of a line and `_` inside an identifier such as `snake_case`, and confirm those literal-entry cases are not intercepted by auto-pairing.
