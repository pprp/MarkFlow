# MF-138 - Export profiles remember targets, themes, and YAML-aware publishing options

## Notes
Typora export supports Export with Previous, overwrite-with-previous, configurable export items, per-file YAML front matter overrides, PDF metadata, and custom header/footer/body content. This selected backlog direction reframes that capability as Export Profiles: reusable publishing targets that remember format, destination, theme, table of contents, math/diagram handling, and metadata options. MarkFlow has basic HTML/PDF/Pandoc export but no verified feature for these reusable publishing workflows.

Sources: https://support.typora.io/Export/ and https://support.typora.io/YAML/

## Steps
- Persist the most recent successful export command and target per document/window.
- Expose Export with Previous and Export and Overwrite with Previous from the desktop menu.
- Add export profile plumbing for format, output directory, export theme, table of contents, math/diagram handling, YAML metadata/variables, PDF metadata, and configurable custom export entries.
- Let users save, update, select, and delete named profiles for repeated publishing workflows.

## Automated Verification
- Prove previous-export state is updated only after a successful export and reused on command.
- Verify YAML title, author, and keywords can flow into HTML title/meta or PDF metadata when enabled.
- Verify a custom export item appears in File > Export and passes configured arguments to Pandoc or the custom command.
- Verify named export profiles can be created, updated, selected, and deleted without losing previous-export state.

## Manual Verification
- Export a document, save the options as a named profile, run Export with Previous, and confirm it reuses the prior target without another target prompt.
- Export a document with YAML metadata and confirm enabled metadata appears in the resulting file.
