# MF-029 - Dragging, pasting, or inserting local images creates markdown image references without breaking asset paths

## Notes
MarkFlow already covers the core local-image insertion slice in `packages/editor/src/editor/MarkFlowEditor.tsx` and `packages/editor/src/editor/extensions/smartPaste.ts`: dragging or pasting an image inserts markdown image syntax at the caret and emits desktop-bridge events for downstream file handling, while `packages/editor/src/editor/decorations/linkDecoration.ts` renders the existing inline preview. Typora's broader image workflow still goes further with documented image-file management and upload actions such as Delete Image, Move/Copy Image To, and Upload Image / Upload All Local Images, which remain outside this verified slice. Sources: https://support.typora.io/Images/ and https://support.typora.io/Upload-Image/.

## Steps
- Open a saved markdown document and insert one or more local images via drag and drop, clipboard paste, or an explicit insert-image action.
- Confirm MarkFlow inserts markdown image references at the caret and shows the existing inline preview when the caret leaves the image syntax.
- If a destination rule for inserted images is configured, confirm the image file is copied and the `src` is rewritten predictably; otherwise confirm the original local path is referenced consistently.

## Automated Verification
- Add focused editor and desktop tests for drag/drop or clipboard image insertion, markdown `![alt](src)` generation, and destination-path rewriting for inserted local assets.

## Manual Verification
- Drag image files from the OS into the desktop app and paste an image from the clipboard, then verify the written markdown path and rendered preview both stay valid after save and reopen.
