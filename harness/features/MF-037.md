# MF-037 - Superscript and subscript syntax renders inline without mutating source

## Notes
BUG (found 2026-04-16, detailed live-preview test): The superscript pattern `/\^([^^\n]+)\^/g` in `packages/editor/src/editor/decorations/inlineDecorations.ts:30` does not exclude footnote references (`[^label]`) and therefore matches across two adjacent references. In the starter document the line `This has a footnote[^1] and another[^2].` renders the literal text ` and another` with `class="mf-superscript"` (vertical-align: super, font-size: 0.75em), because the regex greedily pairs the `^` in `[^1]` with the `^` in `[^2]`. Confirmed via `preview_eval` — `.mf-superscript` span with text ' and another' shows `fontSize: 11.25px` and `verticalAlign: super`. There is also no dedicated test coverage for superscript/subscript (the "not-done" TODO from the notes was never completed); `footnoteDecoration.test.tsx` exercises footnotes alone but never in combination with the superscript extension. Fix should either (a) reject matches that overlap a footnote/LinkReference node, (b) require a non-`^` non-`[` boundary before the opening `^`, or (c) run the footnote replacement first and skip ranges already replaced.

## Steps
- Write inline superscript with `^text^` and subscript with `~text~` syntax in a WYSIWYG document.
- Move the caret away from the marked span and confirm the rendered `<sup>` or `<sub>` widget appears with delimiters hidden.
- Re-enter the span and confirm the raw delimiters reappear and are editable without corruption.

## Automated Verification
- Add `inlineDecorations.test.tsx` cases for superscript and subscript spanning, delimiter-hide behavior, and cursor-reveal round trips.

## Manual Verification
- Write a chemistry formula and a footnote reference using both syntaxes and verify rendered output is legible in WYSIWYG mode.
