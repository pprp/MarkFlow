# MF-133 - Mac detection uses deprecated `navigator.platform`

## Notes
Keyboard shortcut detection in `App.tsx` line 1157 uses `navigator.platform.toUpperCase().indexOf('MAC')` to determine whether the host is macOS. `navigator.platform` is deprecated in modern browsers and may return an empty string in future browser versions, silently breaking all Mac-specific keyboard shortcuts (`Cmd+[`, `Cmd+]`, `` Cmd+` ``, etc.).

The fix is to prefer `navigator.userAgentData?.platform`, keep populated `navigator.platform` as the primary legacy fallback, and only use a desktop-mac `navigator.userAgent` fallback when both platform fields are empty.

Root cause: `App.tsx` line 1157.

2026-04-24T08:57:11+08:00 implementation: `App.tsx` now uses `isMacPlatform()` from `packages/editor/src/platform.ts` instead of reading `navigator.platform` directly. The helper prefers `navigator.userAgentData.platform`, falls back to legacy `navigator.platform` when it is populated, and only then checks for a desktop macOS user agent when both platform fields are empty. Focused tests in `packages/editor/src/__tests__/platform.test.ts` cover user-agent data, an empty `navigator.platform` with a macOS user agent, a mobile user agent that must not be treated as macOS, and Windows remaining on Ctrl shortcut handling, while `packages/editor/src/__tests__/App.test.tsx` proves `Meta+[` and `Meta+]` outline-history shortcuts still work when `navigator.platform` is empty.

## Steps

- Press `Cmd+[` (navigate back) — it does not fire.
