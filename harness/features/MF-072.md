# MF-072 - Image drag-to-resize handles update width/height in markdown source

## Notes
Implemented on 2026-04-16 in `packages/editor/src/editor/decorations/linkDecoration.ts`, `packages/editor/src/editor/__tests__/linkDecoration.test.tsx`, and `packages/editor/src/styles/global.css`. Markdown image widgets now render hover-only corner/edge resize handles in WYSIWYG mode, resize live while dragging, and persist the release size back into markdown as canonical `{width=... height=...}` attributes; the decoration path also recognizes existing Typora-style `=WIDTHxHEIGHT` shorthand on reopen. Automated verification passed with `pnpm --filter @markflow/editor test:run -- src/editor/__tests__/linkDecoration.test.tsx` (the current package script runs the full editor suite), `pnpm --filter @markflow/editor lint`, `pnpm --filter @markflow/editor build`, and `pnpm harness:verify`. `passes` remains false because the required live-app manual drag-resize check across three consecutive images was not truthfully completed in this terminal session. Sources: https://support.typora.io/Images/ and https://support.typora.io/What%27s-New-1.1/.

## Steps
- Render a markdown image in WYSIWYG mode and hover to reveal corner/edge resize handles.
- Drag a handle and confirm the preview resizes live without corrupting markdown syntax.
- Release the handle and confirm the underlying source gains a `{width=...}` / `=WIDTHxHEIGHT` attribute (Typora-compatible) that survives reopen.
