# MF-072 - Image drag-to-resize handles update width/height in markdown source

## Notes
Implemented on 2026-04-16 in `packages/editor/src/editor/decorations/linkDecoration.ts`, `packages/editor/src/editor/__tests__/linkDecoration.test.tsx`, and `packages/editor/src/styles/global.css`. Markdown image widgets now render hover-only corner/edge resize handles in WYSIWYG mode, resize live while dragging, and persist the release size back into markdown as canonical `{width=... height=...}` attributes; the decoration path also recognizes existing Typora-style `=WIDTHxHEIGHT` shorthand on reopen.

Live verification completed on 2026-04-19 against the production renderer preview at `http://localhost:4173` with a temporary three-image fixture served from `http://localhost:4174`. Dragging the southeast resize handle on three consecutive images of `16:9`, `1:1`, and `3:5` aspect ratios persisted `{width=416 height=234}`, `{width=312 height=312}`, and `{width=223 height=372}` into Source mode, and returning to Preview reapplied those exact dimensions to all three image widgets.

For this closeout, the truthful automated verification path was `pnpm --filter @markflow/editor exec vitest run src/editor/__tests__/linkDecoration.test.tsx`, `pnpm --filter @markflow/editor lint`, `pnpm --filter @markflow/editor build`, and `pnpm harness:verify`. The older `pnpm --filter @markflow/editor test:run -- src/editor/__tests__/linkDecoration.test.tsx` alias currently fans out to the entire editor suite and hits an unrelated dirty-tree failure in `src/__tests__/App.test.tsx` (`renders a quieter titlebar by demoting secondary writing modes to icon controls`), so it was recorded as non-feature noise rather than as the truth source for `MF-072` acceptance. Sources: https://support.typora.io/Images/ and https://support.typora.io/What%27s-New-1.1/.

## Steps
- Render a markdown image in WYSIWYG mode and hover to reveal corner/edge resize handles.
- Drag a handle and confirm the preview resizes live without corrupting markdown syntax.
- Release the handle and confirm the underlying source gains a `{width=...}` / `=WIDTHxHEIGHT` attribute (Typora-compatible) that survives reopen.
