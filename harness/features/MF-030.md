# MF-030 - Fenced code blocks display language-aware syntax highlighting in both WYSIWYG and source modes

## Notes
CodeMirror 6 ships `@codemirror/language` and a language-description registry; `StreamLanguage` adapters and `@codemirror/lang-*` packages can be lazy-loaded per fence language identifier. This makes code blocks meaningfully readable for technical writers and engineers, one of MarkFlow's primary target audiences.

## Steps
- Open a document with fenced code blocks annotated with common language identifiers such as `js`, `ts`, `python`, `bash`, and `json`.
- Move the caret outside each block in WYSIWYG mode and confirm token-level syntax colors appear inside the rendered block.
- Enter the block and confirm the source text remains fully editable with the same syntax-highlight colors applied in-place.

## Automated Verification
- Add `codeBlockDecoration.test.ts` cases verifying that language-tagged blocks receive a highlight decoration class and that no-language blocks fall back to plain styling.

## Manual Verification
- Visually confirm at least three language modes (JavaScript, Python, shell) render distinct token colors without corrupting the markdown fence delimiters.
