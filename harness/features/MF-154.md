# MF-154 - Rendered tables support drag-and-drop row and column reordering

## Notes
Typora documents visual table row and column movement through drag handles in table editing workflows. MarkFlow has table rendering, keyboard commands, row movement, and column resizing coverage, but no pointer-driven drag handles that reorder table rows or columns while preserving markdown source. Sources: https://support.typora.io/Table-Editing/ and https://support.typora.io/Markdown-Reference/.

## Steps
- Create a markdown table with multiple rows and columns.
- Render the table in WYSIWYG mode.
- Drag a row handle to reorder rows and confirm the markdown table source updates.
- Drag a column handle to reorder columns and confirm header, separator, and body cells move together.
- Confirm keyboard and source editing behavior remains stable after pointer-driven reordering.

## Automated Verification
- Add focused editor coverage for table drag handle reorder behavior and markdown source updates.
- Run relevant table decoration and table command tests.
- Run `pnpm harness:verify`.

## Manual Verification
- In the editor, drag table row and column handles in a multi-row, multi-column table and confirm visual order plus markdown source order match.
