# MF-154 - Table row and column drag handles reorder markdown tables visually

## Notes
Typora documents visual table row and column movement through drag handles in table editing workflows. MarkFlow currently has table rendering, keyboard commands, row movement, and resizing coverage, but no planned feature for pointer-driven drag handles that reorder table rows or columns while preserving markdown source. Sources: https://support.typora.io/Table-Editing/ and https://support.typora.io/Markdown-Reference/.

## Steps
- Create a markdown table with multiple rows and columns.
- Render the table in WYSIWYG mode.
- Drag a row handle to reorder rows and confirm the markdown table source updates.
- Drag a column handle to reorder columns and confirm header, separator, and body cells move together.
- Confirm keyboard and source editing behavior remains stable after pointer-driven reordering.

## Automated Verification
- Add focused editor coverage for table drag handle reorder command behavior and markdown source updates.
- Run relevant table decoration/command tests and `pnpm harness:verify`.

## Manual Verification
- In the editor, drag table row and column handles in a multi-row, multi-column table and confirm visual order plus markdown source order match.
