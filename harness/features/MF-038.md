# MF-038 - Inline HTML blocks pass through the WYSIWYG renderer without being escaped or stripped

## Notes
Implemented in `packages/editor/src/editor/decorations/inlineHtmlDecoration.ts`, but the current renderer is intentionally limited to a sanitized safe-tag subset (`details`, `summary`, `div`, `span`, `p`, `br`, `strong`, `em`, `b`, `i`, `ul`, `ol`, `li`) with all attributes stripped. That covers the verified collapsible/details-style HTML slice, but it does not provide Typora's broader media/embed support for tags such as `<video>`, `<audio>`, or `<iframe>`. Sources: https://support.typora.io/Media/ and https://support.typora.io/Markdown-Reference/.

## Steps
- Insert raw HTML blocks such as `<details>`, `<summary>`, `<div>`, and plain `<span>` into a markdown document.
- Render in WYSIWYG mode and confirm the HTML is parsed and rendered rather than escaped as literal text.
- Edit the raw HTML source and confirm the rendered output updates and the surrounding markdown is unaffected.

## Automated Verification
- Add decoration tests for the currently supported safe-tag subset, including block containers and plain inline spans with attributes stripped.

## Manual Verification
- Write a `<details>/<summary>` collapsible block plus plain `<div>`/`<span>` HTML and verify they render correctly in WYSIWYG mode without relying on attributes.
