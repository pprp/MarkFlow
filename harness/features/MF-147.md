# MF-147 - Adaptive large-document mode reduces decoration work during typing and fast scrolling

## Notes
MarkFlow already virtualizes decoration work for files over 5,000 lines, but the threshold is static. This feature makes the large-document path adaptive: the editor can temporarily reduce expensive decoration passes while a user types rapidly or scrolls quickly, then complete rich rendering during idle time.

## Steps
- Track document size, visible range size, recent input cadence, and scroll velocity to decide when adaptive large-document mode is active.
- During fast typing or scrolling, render only essential text, headings, links, and simple block structure inside the viewport.
- Restore full decorations for math, diagrams, tables, images, and post-processors after the editor settles.
- Expose a subtle status or diagnostics hook so performance tests can assert which rendering tier is active.

## Automated Verification
- Add editor tests covering adaptive-mode activation by document size, rapid typing, and fast scrolling.
- Add regression tests proving rich decorations return after idle settlement without document mutation.
- Extend performance-budget verification to capture keystroke latency and scroll frame time with adaptive mode enabled.

## Manual Verification
- Open the 180k-line fixture, scroll quickly from top to bottom, type during outline hydration, and confirm the editor stays responsive while rich renderers catch up after idle.
