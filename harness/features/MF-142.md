# MF-142 - Image context actions delete, move, copy, and relink document image assets

## Notes
Typora documents image asset management beyond insertion and upload. Image context actions can delete an image, move an image to a target folder, copy an image to a target folder, and move or copy all referenced images while updating markdown references.

MarkFlow covers insertion, destination preferences, resizing, and PicGo upload, but it does not yet track post-insert file operations for one or all referenced image assets.

Sources: https://support.typora.io/Images/ and https://support.typora.io/Upload-Image/

## Steps
- Add image context/menu actions for delete referenced image, move image to folder, copy image to folder, move all images, and copy all images.
- Update markdown image references after successful move/copy operations, preserving relative-path preferences and YAML image-root behavior.
- Guard destructive delete and move operations with confirmation and surface filesystem errors without mutating the markdown source.

## Automated Verification
- Desktop tests cover delete, move, copy, move-all, copy-all, path rewrite, cancellation, and filesystem failure paths.
- Editor integration tests confirm context actions target the selected or rendered image and preserve resize metadata.

## Manual Verification
- Check local, relative, absolute, and remote image references and confirm only supported local assets are modified.
