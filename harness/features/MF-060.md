# MF-060 - Auto-save writes a recovery checkpoint every 30 seconds without blocking the editor

## Notes
Implemented on 2026-04-15 in `packages/desktop/src/main/fileManager.ts`, `packages/desktop/src/preload/index.ts`, `packages/editor/src/App.tsx`, `packages/desktop/src/main/fileManager.test.ts`, and `packages/editor/src/__tests__/App.test.tsx`. The desktop main process now keeps the latest dirty-document draft in memory, debounces `.markflow-recovery` writes into the system temp directory after 30 seconds of idle time, records clean versus unclean shutdown state, and only surfaces the recovery prompt after an unclean exit. Automated verification passed on 2026-04-15 and again on 2026-04-16 with `pnpm --filter @markflow/desktop test:run -- --grep auto-save` and `pnpm harness:verify`. Manual crash/relaunch verification completed on 2026-04-16 in a live Electron session against `/tmp/markflow-mf060.vh4jMh/recovery.md`: Chromium CDP edited the open file without saving, waited 35 seconds, observed `/var/folders/dl/qdq_vh116gl1yjbd8pxk_bd00000gn/T/.markflow-recovery` containing `checkpoint-from-cdp`, killed the Electron process to preserve `{"cleanExit":false}` in `/Users/pprp/Library/Application Support/@markflow/desktop/.markflow-recovery-session.json`, relaunched MarkFlow, captured the recovery prompt text `Recover the auto-saved changes for recovery.md from 4/16/2026, 7:44:39 AM?` via `Page.javascriptDialogOpening`, accepted it with `Page.handleJavaScriptDialog`, and confirmed the restored checkpoint content plus dirty-state indicator in the renderer.

## Steps
- Edit a document without manually saving.
- Wait 30 seconds and confirm a .markflow-recovery file is written to the system temp directory.
- Simulate a crash; reopen MarkFlow and confirm the recovery prompt appears.
- Accept recovery and confirm the checkpoint content is restored.
