# MF-113 - Windows and Linux desktop packaging ships from CI alongside macOS builds

## Notes
The README pitches "publish anywhere" but only `dist-mac` is produced today. Windows and Linux parity is required before the Phase 4 release cadence can be honest.

## Steps
- Extend the GitHub Actions release workflow to produce Windows `.exe` (NSIS) and Linux `.AppImage`/`.deb` artifacts in addition to the existing macOS `.dmg`.
- Smoke-test each artifact (launch, open a file, toggle WYSIWYG, quit) on its native platform.
- Publish the artifacts to the GitHub release, signed and notarized where applicable.

## Automated Verification
- pnpm --filter @markflow/desktop build

## Manual Verification
- Launch the produced Windows and Linux artifacts and walk through the core editor workflow.
