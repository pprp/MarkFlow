# MF-024 - Horizontal rules render as separators in WYSIWYG mode without losing the underlying markdown

## Notes
Typora documents horizontal rules as core Markdown syntax, and MarkFlow's existing `HorizontalRule` decoration path in `packages/editor/src/editor/decorations/blockquoteDecoration.ts` now has direct automated coverage for both `---` and `***`, including the focus/editability transition and adjacent paragraph preservation. Sources: https://support.typora.io/Markdown-Reference/ and https://support.typora.io/Quick-Start/.

## Steps
- Open markdown containing horizontal rules such as `---` or `***`.
- Move the caret onto and away from the rule line in WYSIWYG mode.
- Confirm a rendered separator appears when unfocused and the original markdown stays editable when focused.

## Automated Verification
- Run `pnpm --filter @markflow/editor test:run -- listAndBlockquoteDecoration.test.tsx` to cover `---` and `***` horizontal rule rendering, caret focus behavior, and adjacent paragraph integrity.
