# MF-132 - Hidden export editor is non-editable so export cannot dirty the active tab

## Notes
Verified on 2026-04-24. The hidden off-screen `MarkFlowEditor` rendered inside `#mf-export-container` now passes `editable={false}` in `packages/editor/src/App.tsx`, so export-time rendering does not expose a writable CodeMirror surface. Focused coverage in `packages/editor/src/__tests__/App.test.tsx` now proves the export editor becomes non-editable while export is in flight, a direct `dispatch()` mutation only affects that hidden instance, the visible editor document stays unchanged, the titlebar dirty indicator never appears, and the export still resolves and tears down `#mf-export-container` cleanly.

## Steps
- Trigger an export (HTML or PDF) from the desktop menu.
- During the export, programmatically dispatch a change to `#mf-export-container .cm-editor` (simulating an accidental event leak).
- Observe no dirty-state change appears on the active tab and the hidden export container is removed after export completes.
