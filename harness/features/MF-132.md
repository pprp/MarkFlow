# MF-132 - Export editor instance is editable and missing `onDocumentEdit`

## Notes
The hidden off-screen `MarkFlowEditor` rendered inside `#mf-export-container` (`App.tsx` lines 2162–2184) does not set `editable={false}`. This means the export instance is a fully editable CodeMirror editor sitting at `left: -9999px`. It also omits `onDocumentEdit` and `onScrollMetricsChange`, so any accidental mutation to the hidden editor goes untracked by the tab state.

Root cause: `App.tsx` lines 2162–2184 — add `editable={false}` and remove callbacks that imply editability.

## Steps
- Trigger an export (HTML or PDF) from the desktop menu.
- During the export, programmatically dispatch a change to `#mf-export-container .cm-editor` (simulating an accidental event leak).
- Observe no dirty-state change appears on the active tab.

## Automated Verification
- Unit test: the export editor component should be instantiated with `editable={false}`.

## Manual Verification
- Trigger an export; the active document's dirty-dot should not appear as a side effect.
