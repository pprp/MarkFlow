# MF-145 - Code block highlighting starts with a slim language pack and loads uncommon languages on demand

## Notes
Language-aware code fences are valuable for MarkFlow's technical-writing audience, but the default CodeMirror language registry can pull many rarely used language chunks into the app. This feature defines a small built-in set for common writing workflows and loads uncommon languages only after a visible fenced block requests them.

## Steps
- Define a default language allowlist for common fences such as JavaScript, TypeScript, JSON, CSS, HTML, Python, Bash, Markdown, and SQL.
- Render common-language code fences with highlighting available immediately.
- Render uncommon-language fences with a lightweight fallback first, then upgrade highlighting after the on-demand language module loads.
- Preserve editable markdown source and avoid layout jumps when a fallback code block upgrades to highlighted output.

## Automated Verification
- Add tests that common languages highlight without invoking the dynamic language loader.
- Add tests that an uncommon language requests its module only when the fenced block is visible.
- Add bundle verification showing rarely used language chunks are excluded from the initial editor entry.

## Manual Verification
- Open a document with common and uncommon fenced languages, inspect the loaded chunks, and confirm uncommon language chunks appear only after their blocks are viewed.
