# MF-086 - Math parity covers code-block math, numbering, and references

## Notes
Typora supports math equation auto-numbering with resolved `\label`/`\ref` pairs and GitHub/GitLab-style fenced code-block math through ` ```math ` fences. Sources: https://support.typora.io/Math/ and https://support.typora.io/Preferences/.

Automated coverage now verifies ` ```math ` fences render through the existing block math widget, preserve markdown source, reveal source while the cursor is inside the fence, and leave ordinary non-math fenced code out of math rendering. Remaining completion gates include equation auto-numbering/reference parity plus the broader line-break and TeX-package parity named in the ledger title.

## Steps
- Write a fenced math block with ` ```math ` and confirm it renders as display math while preserving the underlying fence source.
- Enable math equation auto-numbering and write several display-math blocks with `\label{eq:foo}`.
- Confirm each block renders with an incrementing `(n)` label on the right edge and that the label matches across WYSIWYG and source.
- Reference the label via `\ref{eq:foo}` in prose and confirm the rendered number matches the originating equation.

## Automated Verification

- Add math tests asserting counter increments and reference resolution across multiple equations.
