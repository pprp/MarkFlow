# MF-044 - Highlight spans (`==text==`) render inline without mutating source

## Notes
Implemented in `packages/editor/src/editor/decorations/inlineDecorations.ts` as a document-preserving inline decoration pass that hides `==` delimiters outside the caret, restores raw source when the caret enters the span, and skips inline/fenced code ranges. Automated coverage in `packages/editor/src/editor/__tests__/inlineDecorations.test.tsx` verifies delimiter hiding, caret reveal, code-context exclusion, compatibility with neighboring bold/italic spans, and markdown round trips. Source: https://support.typora.io/Markdown-Reference/.

## Steps
- Write inline highlight markup such as `==important==` in a WYSIWYG document.
- Move the caret away from the span and confirm the delimiter markers hide while the text renders with highlight styling.
- Move the caret back into the span and confirm the raw `==` delimiters reappear for editing without corrupting the underlying markdown.

## Manual Verification
