# MF-117 - Theme hot-reload watches user CSS files and swaps variables without restarting the editor

## Notes
Phase 2 roadmap item. Pairs naturally with the unified preferences dialog (MF-108) and unlocks a community-themes story.

## Steps
- Add a `~/.markflow/themes/` watcher in the desktop main process that emits change events for `.css` files.
- Hot-swap the renderer theme by replacing the injected stylesheet and reapplying CSS variables without reloading the window.
- Expose theme selection (including community themes) from the preferences dialog.

## Automated Verification
- pnpm --filter @markflow/desktop exec vitest run
- pnpm --filter @markflow/editor exec vitest run

## Manual Verification
- Edit a theme CSS file on disk and confirm the running editor updates without restart.
