# MF-164 - Export menu includes built-in HTML without styles output for clean unthemed markup

## Notes
Typora's Export docs include `HTML` and `HTML (without styles)` as separate built-in export targets. MarkFlow already exported styled HTML and PDF, but it did not provide a first-class unthemed HTML export path for clean markup without bundled editor/theme CSS. Sources: https://support.typora.io/Export/

Implemented on 2026-04-24: the desktop File -> Export menu now exposes `HTML (without styles)...`, the shared menu-action union and renderer bridge route that action distinctly, and `serializeRenderedDocumentForExport()` can omit the bundled MarkFlow style block while preserving rendered structure, heading anchors, and YAML-derived title/author/keywords metadata. Previous-export memory now distinguishes styled HTML from HTML-without-styles so replay and overwrite use the correct mode instead of falling back to the styled exporter.

## Steps
- Open a markdown document and choose `File -> Export -> HTML (without styles)...`.
- Save the export and confirm the generated file preserves document structure and YAML-derived metadata without bundled MarkFlow theme/editor CSS.
- Run `Export with Previous` or `Export and Overwrite with Previous` after an unstyled HTML export and confirm the same unstyled mode is replayed.
- Verify the existing styled `HTML...` export still includes the expected CSS and remains unchanged.
