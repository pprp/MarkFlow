# MF-070 - Image insertion honors Typora-style destination/root preferences for dropped and pasted assets

## Notes
MarkFlow already stores YAML metadata and basic image insertion, but does not consume Typora image preference keys like `typora-copy-images-to` / `typora-root-url` during insert flow. Sources: https://support.typora.io/Images/ and https://support.typora.io/YAML/.

## Steps
- Set `typora-copy-images-to` and `typora-root-url` in YAML front matter, then paste/drag local images into the editor.
- Confirm inserted markdown paths are rewritten according to destination/root rules and previews still resolve.
- Repeat with no YAML preferences and confirm fallback insertion behavior matches current MF-029 flow.

## Automated Verification
- Extend image insertion tests in `packages/editor/src/editor/__tests__/smartPaste.test.ts` and `linkDecoration.test.tsx` for YAML-driven destination rewriting and root URL resolution.

## Manual Verification
- Create two notes with different front-matter image preferences and verify each rewrites image paths independently.
