# MF-134 - Closing the last tab briefly sets `activeTabId` to `null`

## Notes
In `handleCloseTab` (`App.tsx` lines 1067–1068`), when the last tab is closed the code calls `replaceTabs([createDocumentTab(null, '')])` to create a new blank tab, then immediately calls `replaceActiveTabId(null)`. A `useEffect` at line 352 detects the mismatch and sets the active tab id asynchronously, but there is one render frame in which `activeTab` is `null` even though a tab exists. Effects and components that depend on `activeTab` receive `null` unexpectedly during this frame.

Root cause: `App.tsx` lines 1067–1068 — `replaceActiveTabId` should receive the new tab's id, not `null`.

## Steps
- Have exactly one tab open.
- Close it.
- If any effect fires synchronously on `activeTab`, observe it receiving `null` rather than the new blank tab.

## Automated Verification
- Unit test: after `handleCloseTab` resolves, `activeTabId` should never be `null` if `tabs.length > 0`.

## Manual Verification
- Close the last open tab; the editor should immediately show a new blank document with no flash of empty/null state.
