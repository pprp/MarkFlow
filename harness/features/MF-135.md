# MF-135 - Global search opens with no explanation when no vault folder is open

## Notes
`GlobalSearch` is keyboard-triggerable (`Mod+Shift+F`) at any time, even when `vaultPath` is `null` (no folder open). The dialog opens but cannot search anything because it needs a folder path. No message is shown to the user explaining why results are unavailable or what they need to do.

Root cause: `App.tsx` lines 2146–2151 — `GlobalSearch` receives `folderPath={vaultPath}` where `vaultPath` may be `null`. The component or its caller should display a friendly prompt to open a folder.

## Steps
- Launch the app without opening a folder.
- Press `Cmd/Ctrl+Shift+F` to open global search.
- Type a search query — no results appear and no explanation is shown.

## Automated Verification
- Unit test: when `folderPath` is `null`, `GlobalSearch` should render an informational message prompting the user to open a folder.

## Manual Verification
- Without a vault open, triggering global search shows a clear "Open a folder to search across files" message.
- After opening a folder, global search works normally.
