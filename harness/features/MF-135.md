# MF-135 - Global search opens with no explanation when no vault folder is open

## Notes
The existing dirty slice already implemented the fix in `packages/editor/src/components/GlobalSearch.tsx` by giving the no-folder empty state higher priority than the normal query/search/no-results branches and by hiding stale grouped results whenever `folderPath` becomes `null`. `GlobalSearch` now keeps a clear `Open a folder to search across files` explanation visible whenever no folder is open, even if a non-empty query string and prior search matches are still present in component state.

Root cause: the component's empty-state rendering treated `query.trim()` as the top-level branch selector, so a preserved non-empty query with `folderPath === null` fell through to `No results found` instead of the no-folder explanation. The rendered result groups also continued to read directly from `results`, so stale file groups could remain visible after the folder disappeared.

This assignment kept the implementation as-is, reran the focused verification successfully, and refreshed the row to `status=verified`, `passes=true`, and `lastVerifiedAt=2026-04-25T16:02:05+0800`. No edits were needed in `packages/editor/src/components/GlobalSearch.tsx` or `packages/editor/src/components/GlobalSearch.test.tsx`.

## Steps
- Launch the app without opening a folder.
- Press `Cmd/Ctrl+Shift+F` to open global search.
- Confirm the dialog explains that a folder must be opened first.
- With a preserved non-empty query state and no folder open, confirm the same explanation remains visible instead of `No results found`.
