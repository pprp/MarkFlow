# MF-098 - HTML comments and HTML entities are preserved in WYSIWYG mode without leaking raw markup

## Notes
Typora documents support for HTML comments and HTML entities, and MarkFlow now preserves that behavior in `packages/editor/src/editor/decorations/inlineHtmlDecoration.ts` by extending the existing HTML replacement path to markdown `CommentBlock`, inline `Comment`, and `Entity` nodes. Focused regression coverage in `packages/editor/src/editor/__tests__/inlineHtmlDecoration.test.ts` verifies block comments stay hidden away from the caret, inline comments do not leak raw markup, entities render as decoded characters, and each range reveals its original source when the caret enters it. Verified on 2026-04-15 with `pnpm --filter @markflow/editor test:run -- src/editor/__tests__/inlineHtmlDecoration.test.ts` plus `node scripts/harness/verify.mjs`. Sources: https://support.typora.io/HTML/ and https://support.typora.io/Markdown-Reference/.

## Steps
- Insert HTML comments such as `<!-- hidden note -->` and HTML entities such as `&copy;`, `&reg;`, or `&nbsp;` into a markdown document.
- Render in WYSIWYG mode and confirm comments do not surface as raw literal markup while entities display with their rendered character semantics.
- Move the caret back into the affected ranges and confirm the underlying source remains editable without corrupting surrounding markdown.
