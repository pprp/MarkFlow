# MF-108 - Unified preferences dialog consolidates editor, theme, export, spellcheck, and launch settings

## Notes
Preferences are currently scattered across `statisticsPreferences.ts`, `markdownMode.ts`, `spellCheckProfile.ts`, `themeManager.ts`, and desktop launch options. Consolidating behind a typed preferences store in `@markflow/shared` unblocks MF-082 and reduces renderer-side drift.

## Steps
- Open Preferences from the application menu; confirm a single dialog exposes editor behavior (markdown mode, statistics, smart input), appearance (theme, dark mode, heading numbering), export defaults, spellcheck language, and launch behavior.
- Change each setting and confirm it persists across app restart and is reflected by the corresponding feature (e.g. strict mode, dark theme, launch behavior).
- Confirm the dialog is fully keyboard navigable and reachable via the command palette.

## Automated Verification
- pnpm --filter @markflow/editor exec vitest run
- pnpm --filter @markflow/desktop exec vitest run

## Manual Verification
- Exercise every preference group on macOS and confirm persistence across restart.
