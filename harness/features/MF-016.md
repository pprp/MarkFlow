# MF-016 - Plugins can register markdown post-processors for custom render transforms

## Notes
Fully implemented: MarkFlowPluginHost in packages/shared/src/index.ts provides registerMarkdownPostProcessor, load, unload, setPlugins, and runMarkdownPostProcessors. The markdownPostProcessorExtension in packages/editor/src/editor/extensions/markdownPostProcessor.ts hooks the plugin host into CodeMirror via ViewPlugin. A sample plugin (createExternalLinkBadgePlugin in packages/editor/src/plugins/externalLinkBadgePlugin.ts) demonstrates the API by classifying external links. App.tsx wires the plugin host into the editor on startup. Two automated tests cover plugin lifecycle (register/run/unload) and source text preservation.

## Steps
- Define a plugin registration API.
- Load a sample plugin that transforms markdown output.
- Confirm the plugin can add a render behavior without breaking core editing.

## Automated Verification
- pnpm --filter @markflow/editor test:run -- src/editor/__tests__/markdownPostProcessor.test.tsx
