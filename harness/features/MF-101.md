# MF-101 - Delete range commands remove word, sentence or line, block, and styled scope with Typora-style semantics

## Notes
Implemented on 2026-04-16 in `packages/editor/src/editor/extensions/deleteRange.ts`, `packages/editor/src/editor/MarkFlowEditor.tsx`, `packages/editor/src/App.tsx`, `packages/editor/src/editor/clearFormatting.ts`, `packages/editor/src/editor/extensions/smartInput.ts`, `packages/editor/src/editor/__tests__/deleteRange.test.ts`, and `packages/editor/src/editor/__tests__/MarkFlowEditor.test.tsx`. This run adds a dedicated delete-range command layer for Typora-style delete-word, delete-line-or-sentence, delete-block, and delete-styled-scope semantics: delete-word now removes the current lexical word while preserving adjacent markdown, delete-line-or-sentence dispatches by block type (sentence in prose, row in tables, source line in code or math blocks), delete-block removes the surrounding heading/paragraph/list-item or fenced block range, and delete-styled-scope drops the innermost inline wrapper under the caret. The command palette now exposes all four commands, while the editor binds `Mod-Shift-D` for delete word, `Mod-Shift-Backspace` for delete line or sentence, and `Mod-Shift-E` for delete styled scope. A tiny prerequisite fix moved `MF-079` duplication off `Mod-Shift-D` onto `Shift-Alt-ArrowDown` so the Typora delete-word shortcut could land without conflict. Automated verification passed again on 2026-04-16 with `pnpm --filter @markflow/editor exec vitest run src/editor/__tests__/deleteRange.test.ts src/editor/__tests__/MarkFlowEditor.test.tsx src/__tests__/App.test.tsx`, `pnpm --filter @markflow/editor lint`, `pnpm --filter @markflow/editor build`, and `pnpm harness:verify`, so the ledger status is now aligned to `ready`. `passes` remains false because the required mixed-document manual shortcut verification was not performed in this terminal session. Sources: https://support.typora.io/Delete-Range/ and https://support.typora.io/Shortcut-Keys/.

## Steps
- Trigger delete-word with the caret inside a word and confirm the current word is removed without corrupting adjacent markdown.
- Trigger delete line or sentence inside prose, code blocks, math blocks, and tables and confirm the deletion respects the current block type instead of blindly deleting raw lines.
- Trigger delete block or styled-scope on headings, list items, or inline styled spans and confirm only the intended markdown scope is removed.
