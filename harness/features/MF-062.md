# MF-062 - Folding collapses any block element (heading section, code block, list) to a single summary line

## Notes
Implemented on 2026-04-16 in `packages/editor/src/editor/extensions/headingFold.ts`, `packages/editor/src/editor/foldingState.ts`, `packages/editor/src/editor/MarkFlowEditor.tsx`, `packages/editor/src/App.tsx`, `packages/desktop/src/main/fileManager.ts`, `packages/desktop/src/preload/index.ts`, `packages/shared/src/index.ts`, `packages/editor/src/editor/__tests__/folding.test.tsx`, `packages/editor/src/__tests__/App.test.tsx`, and `packages/desktop/src/main/fileManager.test.ts`. The fold gutter now collapses heading sections through the next same-level heading plus multiline lists and fenced code blocks to a single summary line, and successful saves persist the active folded ranges into a `*.folds` sidecar that is restored when the file reopens. Automated verification passed on 2026-04-16 with `pnpm --filter @markflow/editor test:run -- --grep folding`, `pnpm --filter @markflow/editor lint`, `pnpm --filter @markflow/editor build`, `pnpm --filter @markflow/desktop exec vitest run src/main/fileManager.test.ts`, `pnpm --filter @markflow/desktop build`, and `pnpm harness:verify`. Manual verification also passed on 2026-04-16 in a live Electron session against `/tmp/markflow-mf062.AisuVg/folding.md`: two heading sections were folded from the gutter, `save-file` wrote `/tmp/markflow-mf062.AisuVg/folding.md.folds` containing `[7,18,26,86]`, and after relaunch the same two fold placeholders remained visible while the unfurled `Gamma` section stayed expanded.

## Steps
- Click the fold gutter icon next to a heading.
- Confirm all lines under that heading until the next same-level heading are hidden.
- Click the icon again and confirm the section expands.
- Confirm folded state survives a file save and reload.
