# MF-105 - Enter creates a new paragraph while Shift+Enter inserts a single line break with Typora-style source mapping

## Notes
Typora's Markdown Reference and Line Break docs explicitly distinguish `Return` for a new paragraph from `Shift+Return` for a single line break, with source mode showing a blank line between paragraphs. The shipped editor behavior in `packages/editor/src/editor/extensions/smartInput.ts` is now locked by both keymap-level tests and real MarkFlowEditor DOM keydown coverage in `packages/editor/src/editor/__tests__/smartInput.test.ts` and `packages/editor/src/editor/__tests__/MarkFlowEditor.test.tsx`, including plain-paragraph Enter, Shift+Enter, source/split raw Enter behavior, and unordered/ordered/task list continuation plus empty-item exit so the paragraph-break feature cannot silently regress list handling. Automated verification passed on 2026-04-16 with the listed vitest, lint, build, and harness verification commands, so this feature can be truthfully marked verified without a separate desktop-only gate. Sources: https://support.typora.io/Markdown-Reference/ and https://support.typora.io/Line-Break/.

## Steps
- In WYSIWYG mode, place the caret at the end of a plain paragraph and press Enter; confirm the editor starts a new paragraph instead of inserting a lone source newline.
- Switch to source mode after that edit and confirm the markdown now contains a blank line between the two paragraphs.
- In WYSIWYG mode, press Shift+Enter inside a paragraph and confirm the editor inserts a single visible line break while keeping the text in the same paragraph when rendered.
