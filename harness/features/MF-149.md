# MF-149 - Writer mode starts with core editing modules and enables heavy capabilities on demand

## Notes
MarkFlow can serve both lightweight writing and power-user publishing workflows. Writer mode makes the default startup path smaller by enabling only core editing, file IO, basic markdown decorations, search, and theme support, while heavier capabilities such as collaboration, plugins, math, diagrams, and advanced export load when the user turns them on or opens content that needs them.

## Steps
- Add a Writer/Core mode preference that defaults to the lean startup profile for new installs.
- Keep essential editing, file open/save, source toggle, basic markdown WYSIWYG, and search available immediately.
- Load math, diagrams, collaboration, plugin post-processors, and advanced export modules only when enabled or required by visible content.
- Make mode changes reversible without requiring restart when practical, and otherwise surface a clear restart requirement.

## Automated Verification
- Add preference and App integration tests for mode hydration, persistence, and module enablement.
- Add build or runtime tests proving collaboration, plugin, math, and diagram modules are absent from the lean startup path.
- Re-run editor smoke tests in both Writer mode and full capability mode.

## Manual Verification
- Launch a fresh app profile, confirm ordinary editing works before heavy chunks load, then enable advanced capabilities and confirm the same document renders math, diagrams, and plugins correctly.
