# MF-148 - Symbol table indexing runs in a real Web Worker for large documents

## Notes
The current background indexer yields through the main thread and avoids many full-document hot paths, but very large documents still compete with editing for CPU time. This feature moves heading and anchor indexing for large documents into a dedicated Web Worker while preserving the existing synchronous fallback for tests and constrained environments.

## Steps
- Create a symbol-table worker that accepts document snapshots or incremental content payloads and returns headings plus anchor positions.
- Route large-document indexing through the worker and keep small documents on the current lightweight path.
- Drop stale worker results when the active document, markdown mode, or content version changes.
- Preserve internal link navigation, outline updates, and TOC decoration behavior while indexing happens off the main thread.

## Automated Verification
- Add worker tests for heading extraction, anchor deduplication, stale-result dropping, and markdown-mode differences.
- Add editor integration tests proving large documents use the worker path and small documents keep the direct path.
- Re-run existing outline, indexer, TOC, and navigation tests.

## Manual Verification
- Open the 180k-line fixture in Electron, type while the outline hydrates, and confirm input remains responsive until the full symbol table arrives.
