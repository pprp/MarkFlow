# MF-025 - YAML front matter remains intact and editable as document metadata

## Notes
Implemented in `packages/editor/src/editor/decorations/yamlFrontMatter.ts` and covered by `packages/editor/src/editor/__tests__/yamlFrontMatter.test.ts`, which verifies top-of-file detection, `---` / `...` fences, and document-preserving decorations. The current verified scope is preservation and visual distinction of YAML metadata in the editor; MarkFlow does not yet consume Typora-specific keys such as `typora-root-url` or `typora-copy-images-to` for export or image workflows. Sources: https://support.typora.io/YAML/ and https://support.typora.io/media/export/Markdown-Reference.pdf.

## Steps
- Open a markdown document containing YAML front matter delimited by `---`.
- Edit both metadata fields and normal body content while switching between source and WYSIWYG mode.
- Confirm the front matter block remains intact and no keys, delimiters, or body content are corrupted.

## Automated Verification
- pnpm --filter @markflow/editor test:run -- src/editor/__tests__/yamlFrontMatter.test.ts

## Manual Verification
- Optional manual spot-check: edit `title`, `tags`, `typora-root-url`, and `typora-copy-images-to` keys while switching modes, and confirm the YAML block remains unchanged.
