# MF-081 - Image upload integration handles pasted/dropped, selected, all-local, and YAML-triggered Typora upload flows

## Notes
Implemented on 2026-04-16 in `packages/desktop/src/main/imageUploadManager.ts`, `packages/desktop/src/preload/index.ts`, `packages/editor/src/App.tsx`, `packages/editor/src/editor/MarkFlowEditor.tsx`, and `packages/editor/src/editor/extensions/smartPaste.ts`. MarkFlow now persists PicGo-core/custom-command upload preferences, materializes pasted/dropped images into a managed local asset path, auto-runs uploader commands with the image path appended, rewrites markdown to the returned remote URL on success, and preserves the local asset with an error toast on failure. Focused coverage in `packages/desktop/src/main/imageUploadManager.test.ts` verifies success/failure/timeout, while `packages/editor/src/__tests__/App.test.tsx` verifies the URL rewrite and failure-fallback paths. Automated verification passed on 2026-04-16 via the targeted desktop/editor vitest runs, `pnpm --filter @markflow/shared lint`, `pnpm --filter @markflow/shared build`, `pnpm --filter @markflow/editor lint`, `pnpm --filter @markflow/editor build`, `pnpm --filter @markflow/desktop lint`, `pnpm --filter @markflow/desktop build`, and `pnpm harness:verify`. `passes` remains false because the required real PicGo manual upload verification was not performed in this terminal session. Sources: https://support.typora.io/Upload-Image/ and https://github.com/PicGo/PicGo-Core.

Slice note (2026-04-23): Added a command-palette "Upload Selected Image" path for an existing selected local markdown image reference. The command resolves relative/absolute/file URL image sources against the current markdown file, calls the existing desktop upload pipeline directly without re-ingesting the file, and rewrites the selected markdown image to the returned remote URL. Follow-up hardening captures the original tab id, selection range, and selected markdown before upload, then rewrites that stable range only if the expected text is still present, so async completion cannot overwrite a later selection. Manual selected-image upload now passes `manual: true`, allowing configured uploaders to run even when automatic paste/drop upload is disabled while preserving the auto-upload gate for inserted images.

Remaining residual: selected-image upload is now automated for a selected local markdown image reference, including async selection-change and manual-with-auto-disabled cases, but all-local-image upload, YAML-triggered upload rules, rendered-widget/context-menu ergonomics, and real PicGo/manual remote-store verification remain open.

## Steps

- Simulate an upload failure and confirm the editor surfaces an error toast without losing the pasted asset.
