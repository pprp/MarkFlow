# MF-035 - Split view shows live source and rendered preview side-by-side and keeps them synchronized

## Notes
Split view is a power-user mode between pure WYSIWYG and pure source editing. The preview pane can reuse the existing markdown rendering pipeline (decorations applied to a read-only CodeMirror view, or a simple `marked`/`remark` HTML output into an iframe) without duplicating the core editor. Particularly useful when writing technical docs that rely heavily on math, diagrams, or tables.

## Steps
- Toggle split view from the toolbar or command palette and confirm the window divides into a source pane on the left and a rendered preview pane on the right.
- Edit markdown in the source pane and confirm the preview updates in near-real-time.
- Scroll in either pane and confirm the other pane synchronizes scroll position proportionally.

## Automated Verification
- Add component tests verifying the split layout renders, that source mutations trigger preview updates within one render cycle, and that pane-resize handles adjust layout correctly.

## Manual Verification
- Open a long document in split mode and confirm scroll sync, inline math, and mermaid diagrams render correctly in the preview pane.
