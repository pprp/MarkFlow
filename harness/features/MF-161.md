# MF-161 - Native Share actions expose selected text, images, and files

## Notes
Typora's macOS support docs describe `File` -> `Share` for the current file, and Typora 1.12 restored a context-menu Share action for selected text or images across supported macOS versions. MarkFlow currently has copy/export actions and file-manager reveal/open flows, but no native share-sheet bridge for selected editor content, rendered images, or the active saved file. Sources: https://support.typora.io/Typora-on-macOS/ and https://support.typora.io/What%27s-New-1.12/.

## Steps
- Open a saved markdown file and invoke `File` -> `Share`.
- Select rendered text in the editor, open the context menu, and invoke Share.
- Select or right-click a rendered image, open the context menu, and invoke Share.
- Confirm the native OS share sheet receives the expected file, plain text, or image payload without exposing markdown syntax markers.
- Confirm the Share action is disabled or explains the limitation for unsaved files and unavailable OS share targets.

## Automated Verification
- Add desktop menu/context-menu tests proving Share is present only when the platform and selection/file state support it.
- Add renderer bridge tests proving selected text and image payloads are serialized without markdown marker leakage.
- Run relevant desktop/editor tests and `pnpm harness:verify`.

## Manual Verification
- On macOS, share a saved markdown file, selected rendered text, and a rendered image to at least one native target such as Notes, Mail, or Messages.
