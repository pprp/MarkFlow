# MF-153 - Raw HTML styling attributes render safely without stripping supported inline styles

## Notes
Typora documents raw HTML styling such as inline `style` attributes and broader HTML support in Markdown documents. MarkFlow currently renders the safe-tag subset from MF-038 but strips attributes, so styled spans and related inline styling are not preserved in WYSIWYG rendering. Sources: https://support.typora.io/Markdown-Reference/ and https://support.typora.io/HTML/.

## Steps
- Create markdown containing inline HTML such as `<span style="color:red">red text</span>` and safe non-style attributes where supported.
- Render in WYSIWYG mode.
- Confirm supported style attributes affect only the intended HTML content.
- Confirm unsafe attributes or scriptable content are still stripped or inert.

## Automated Verification
- Add focused editor coverage for safe inline HTML style rendering and unsafe attribute sanitization.
- Run the relevant inline HTML decoration tests and `pnpm harness:verify`.
- Verification note (2026-04-23): Implemented a constrained inline `style` sanitizer for raw HTML decorations in `packages/editor/src/editor/decorations/inlineHtmlDecoration.ts`. The sanitizer preserves only an explicit safe CSS property/value subset such as `color`, `background-color`, and `font-weight`, while stripping event handlers, unsupported attributes, script tags, URL-bearing CSS, `expression(...)`, and unsupported CSS properties. Focused red-first coverage was added in `packages/editor/src/editor/__tests__/inlineHtmlDecoration.test.ts`; it failed before the sanitizer preserved style attributes and passed after the implementation. Reapplied verification passed with `pnpm --filter @markflow/editor exec vitest run src/editor/__tests__/inlineHtmlDecoration.test.ts`, `pnpm harness:verify`, and `git diff --check`.
