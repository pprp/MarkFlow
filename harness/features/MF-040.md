# MF-040 - Wikilinks (`[[page]]`) resolve to internal document links and navigate within the active vault

## Notes
Wikilink syntax is the cornerstone of Obsidian and Roam-style networked note-taking, making MarkFlow competitive in the knowledge-management space. Implementing wikilink decoration in `linkDecoration.ts` and resolving targets via the desktop file tree covers the Typora/Obsidian overlap audience and sets the foundation for backlinks (a Phase 4 feature).

## Steps
- Type `[[filename]]` or `[[filename|display text]]` in a document inside an open vault folder.
- Move the caret out of the wikilink and confirm a rendered link widget appears.
- Cmd/Ctrl+click the widget and confirm the referenced document opens (or a new-file prompt appears if it doesn't exist).

## Automated Verification
- Add decoration tests for wikilink parsing, alias rendering, broken-link visual state, and click-dispatch to the file-open IPC path.

## Manual Verification
- Create a two-file vault, write wikilinks between them, and verify bidirectional navigation works without corrupting either file's source.
