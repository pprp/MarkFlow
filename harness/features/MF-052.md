# MF-052 - Chunk-based file loader streams large files in 64 KB chunks to avoid blocking the main thread on open

## Notes
Verified on 2026-04-12. Chunk-based file loader in `fileManager.ts` uses `fs.createReadStream` with 64 KB chunks for files >1 MB, emits `file-loading-progress` IPC with live preview, final `file-opened` sent after full assembly. Shared types, preload, and App.tsx render progress as large-file preview panel. 200 MB fixture generated via `pnpm harness:fixture:large --megabytes 200`. All desktop/editor tests pass. Not-tested: manual open-time/OOM check on 200 MB file.

## Steps
- Open a 200 MB markdown file via File > Open.
- Confirm the first screen-full of content is visible within 1 second.
- Confirm a progress indicator is shown until the full file is loaded.
- Confirm no OOM crash occurs on a machine with 8 GB RAM.

## Automated Verification
- pnpm --filter @markflow/desktop test:run -- --grep chunk-loader

## Manual Verification
- Run `pnpm harness:fixture:large -- --megabytes 200 --output harness/fixtures/mf-large-200mb.md`, open that file, and measure time-to-first-line with a stopwatch.
