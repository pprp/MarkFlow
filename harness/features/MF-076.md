# MF-076 - Paste as plain text shortcut strips rich formatting before insertion

## Notes
- Implemented on 2026-04-15 in `packages/editor/src/editor/extensions/smartPaste.ts` by tracking a short-lived `Cmd/Ctrl+Shift+V` intent window so the next paste prefers `text/plain` even when the clipboard also carries HTML, while preserving the default HTML-to-Markdown smart-paste path and existing image-paste priority.
- Desktop menu coverage was added on 2026-04-21 so the app-level `Cmd/Ctrl+Shift+V` accelerator invokes Electron's native `pasteAndMatchStyle` role instead of relying only on renderer keydown delivery.
- Coverage lives in `packages/editor/src/editor/__tests__/smartPaste.test.ts` and currently verifies default rich-text conversion, both `Ctrl+Shift+V` and `Cmd+Shift+V` plain-text shortcut paths, one-shot shortcut reset, blur reset, 1-second timeout expiry, and image-paste precedence.
- Desktop menu coverage lives in `packages/desktop/src/main/menu.test.ts` and verifies the `Paste as Plain Text` menu item exposes `CmdOrCtrl+Shift+V` with the native `pasteAndMatchStyle` role.
- Source references used during implementation: https://support.typora.io/Copy-Paste/ and https://support.typora.io/Shortcut-Keys/.
- Ledger truth remains: `status=ready`, `passes=false`, `lastVerifiedAt=null`.
