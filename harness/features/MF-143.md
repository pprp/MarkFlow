# MF-143 - Export pipeline renders the active document to a Typora-style image

## Notes
Typora exposes Image as a built-in export format alongside PDF and HTML. Its image export supports configurable output width, font size, image quality, and export theme for social-media-style document snapshots.

MarkFlow currently tracks desktop HTML/PDF export, Pandoc-backed formats, export presets, and page-break behavior, but the ledger did not yet cover exporting the rendered active document as an image.

Sources: https://support.typora.io/Export/ and https://support.typora.io/Markdown-Reference/

## Steps
- Add an Image export target under the desktop export menu for the active document.
- Render the document with export-specific width, font-size, quality, and theme options.
- Surface export failure when a document is too long or rendering cannot complete without leaving a partial success state.

## Automated Verification
- Desktop tests cover image export menu wiring, save-path handling, export success, cancellation, and renderer failure reporting.
- Editor or integration tests cover export rendering options for width, font size, quality, and theme selection.

## Manual Verification
- Export a short document with headings, tables, images, code, and math to an image and inspect that the rendered output matches the selected theme and size options.
- Try an intentionally long document and confirm the app reports a clear failure instead of producing a misleading partial image.
