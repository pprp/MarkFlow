# MF-034 - Admonition and callout blocks render with distinct icons and colors while preserving blockquote markdown

## Notes
Typora added GitHub-style alert/callout rendering in Typora 1.8 and its Markdown Reference now lists the same five blockquote forms (`NOTE`, `TIP`, `IMPORTANT`, `WARNING`, `CAUTION`). MarkFlow currently styles generic blockquotes and horizontal rules only, so this remains a real parity gap that can be implemented as a specialization of the existing `blockquoteDecoration.ts` path. Sources: https://support.typora.io/What%27s-New-1.8/ and https://support.typora.io/Markdown-Reference/.

## Steps
- Write blockquotes using GitHub-style callout syntax such as `> [!NOTE]`, `> [!WARNING]`, `> [!TIP]`, `> [!IMPORTANT]`, and `> [!CAUTION]`.
- Move the caret out of each callout block in WYSIWYG mode and confirm a styled, icon-labelled callout widget replaces the raw blockquote.
- Edit the callout text and confirm the type label, the `> ` prefix markers, and the body content all remain intact in the underlying markdown.

## Automated Verification
- Add `blockquoteDecoration.test.ts` cases for each callout type, verifying icon class, color class, and source round-trip fidelity.

## Manual Verification
- Visually confirm each callout type renders a distinct icon and accent color, and that switching between types updates the rendered widget without breaking the markdown.
