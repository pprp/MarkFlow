# MF-129 - Outline panel collapse state is not persisted across sessions

## Notes
`outlineCollapsed` is plain React state initialised to `false` with no localStorage persistence. Every page/app reload expands the outline panel regardless of what the user chose last session.

Root cause: `App.tsx` line 252 — `useState(false)` with no persistence hook, unlike `headingNumberingEnabled` and `sourceLineNumbersEnabled` which both have load/persist helpers.

## Steps
- Open a document with headings so the outline toggle is visible.
- Click the outline toggle to collapse the outline panel.
- Reload the page / relaunch the app.
- Observe the outline panel is expanded again.

## Automated Verification
- Unit test: after collapsing, `localStorage` should contain the preference; on next mount the panel should initialise as collapsed.

## Manual Verification
- Collapse the outline, reload, confirm it stays collapsed.
- Expand the outline, reload, confirm it stays expanded.
