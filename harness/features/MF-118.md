# MF-118 - Plugin API beta exposes commands, markdown post-processors, and custom views

## Notes
Phase 3 roadmap cornerstone. Validating the shape with a real example plugin avoids designing the API in the abstract.

## Steps
- Promote `packages/editor/src/editor/extensions/markdownPostProcessor.ts` into a documented public API with `registerMarkdownPostProcessor`, `registerCommand`, and `registerView` entry points.
- Ship one reference plugin (for example a word-count badge) under `packages/plugin-examples/` and load it from a plugin directory.
- Document the API surface, lifecycle, and security boundaries in `docs/plugin-api.md`.

## Automated Verification
- pnpm --filter @markflow/editor exec vitest run
- pnpm --filter @markflow/editor build

## Manual Verification
- Load the reference plugin in the desktop app and confirm it activates and deactivates cleanly.
