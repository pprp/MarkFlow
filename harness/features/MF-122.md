# MF-122 - Rendered markdown table cells wrap long and multiline content instead of eliding it

## Notes
User-reported issue on 2026-04-21: long or multiline table content is currently hidden with ellipsis in rendered/WYSIWYG tables. Table rows should expand vertically and wrap content so users can read and edit the full cell text without switching out of the rendered table view.

## Steps
- Open a markdown table containing a very long sentence in one cell and enough neighboring columns to force constrained widths.
- Open a table containing explicit multiline cell content supported by the editor's markdown table model.
- Render the table in WYSIWYG mode and confirm cells wrap naturally, rows grow vertically, and no content is truncated with ellipsis.
- Confirm editing, column resizing, source/WYSIWYG toggling, undo, and redo preserve the full table content.

## Automated Verification
- Add or update table decoration tests that assert wrapped table cells do not rely on `white-space: nowrap`, `overflow: hidden`, or `text-overflow: ellipsis` for normal content.
- Add regression coverage for long table cell content staying visible and editable after mode toggles.

## Manual Verification
- In the live editor, paste a table with long and multiline cell content, resize the window to a narrow width, and confirm every cell remains visible through automatic wrapping/row expansion.
