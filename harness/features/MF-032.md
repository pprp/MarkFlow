# MF-032 - Auto-save periodically writes the live buffer to disk so no work is lost on unexpected quit

## Notes
Typora auto-saves on configurable intervals and offers crash recovery. The desktop `fileManager.ts` already has async write logic; adding a debounced auto-save loop driven by CodeMirror `docChanged` events is a natural extension. Reduces the most common user data-loss scenario.

## Steps
- Open and edit a named file and let the auto-save interval elapse without explicitly saving.
- Force-quit the app and reopen it; confirm the last auto-saved content is present.
- Confirm the interval and enable/disable toggle are configurable in preferences and that untitled buffers are written to a recovery location.

## Automated Verification
- Add desktop integration tests that spy on the file-write path and assert it fires within the configured interval after document mutations.

## Manual Verification
- Edit a file for 30 seconds without saving, kill the process, reopen, and verify content recovery.
