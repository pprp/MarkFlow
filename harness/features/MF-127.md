# MF-127 - `toggleViewMode` (Mod+/) jumps to WYSIWYG instead of Source when in Reading or Split mode

## Notes
Verified on 2026-04-22. `toggleViewMode` in `App.tsx` now maps Source to Preview, Preview and Reading to Source, and leaves Split unchanged. The command palette description now reflects the active mode so Reading advertises "Switch to source mode" and Split advertises that Split remains active.

Original root cause: `App.tsx` used `m === 'wysiwyg' ? 'source' : 'wysiwyg'`, so Reading and Split both jumped to Preview through the generic non-WYSIWYG branch.

## Steps
- Switch to Reading mode via the segmented control.
- Press `Mod+/` (`Ctrl+/` on Windows/Linux) and confirm the active segment switches to Source.
- Press `Mod+/` again from Source and confirm the active segment switches back to Preview.
- Switch to Split mode, press `Mod+/`, and confirm Split remains active.
