# MF-127 - `toggleViewMode` (Mod+/) jumps to WYSIWYG instead of Source when in Reading or Split mode

## Notes
`toggleViewMode` in `App.tsx` is implemented as `m === 'wysiwyg' ? 'source' : 'wysiwyg'`. When the user is in Reading or Split mode and presses `Mod+/`, the intent is to access the source view, but the shortcut instead switches to WYSIWYG (preview). This is surprising and inconsistent with the label "toggle source / preview".

Root cause: `App.tsx` line 569.

## Steps
- Switch to Reading mode via the segmented control.
- Press `Mod+/` (Ctrl+/ on Windows/Linux).
- Observe the mode switches to Preview (WYSIWYG), not Source.
- Repeat from Split mode — same result.

## Automated Verification
- Unit test: calling `toggleViewMode` from `'reading'` should produce `'source'`, and calling it from `'source'` should produce `'wysiwyg'`.

## Manual Verification
- From Reading mode, pressing `Mod+/` should switch to Source mode.
- Pressing `Mod+/` again from Source mode should switch back to WYSIWYG.
- Split mode is unaffected by `Mod+/`.
